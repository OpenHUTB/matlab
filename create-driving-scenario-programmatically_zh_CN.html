以编程方式创建驾驶场景

Open Script

此示例演示如何为合成传感器数据和跟踪算法生成真实情况。它还展示了如何在开环和闭环模拟中更新演员姿势。最后，它展示了如何使用驾驶场景执行坐标转换并将其合并到鸟瞰图中。

在此示例中，您将以编程方式从 MATLAB® 命令行创建驾驶方案。或者，您可以使用驾驶方案设计器应用以交互方式创建方案。有关示例，请参阅以交互方式创建驾驶方案和生成合成传感器数据。

介绍

驾驶场景的目标之一是生成“真实”测试用例，用于特定车辆上使用的传感器检测和跟踪算法。

此基本事实通常在全局坐标系中定义;但是，由于传感器通常安装在移动的车辆上，因此需要将这些数据转换为随车辆移动的参考系。驾驶方案会自动促进此转换，允许您在全局坐标中指定对象的道路和轨迹，并提供用于在场景中任何参与者的参考系中转换和可视化此信息的工具。

将姿势信息转换为参与者的参考帧

驾驶场景由道路和可移动对象的模型组成，称为参与者。您可以使用Actor对场景中的行人、停车计时器、消防栓和其他对象进行建模。Actor由具有长度，宽度，高度和雷达横截面（RCS）的长方体组成。演员围绕其底面中心的单个点进行定位和定向。

一种特殊的在轮子上移动的参与者是车辆，它定位和定向在后轴中心正下方的地面上，这是一个更自然的旋转中心。

所有Actor（包括载具）都可以通过指定各自的位置、横滚、俯仰、偏航、速度和角度速度属性来放置在场景中的任何位置。

下面是一个场景示例，该场景由相距 10 米的两辆车组成，分别以每秒 3 米和 4 米的速度驶向原点：

scenario = drivingScenario;
v1 = vehicle(scenario,'ClassID',1','Position',[6 0 0],'Velocity',[-3 0 0],'Yaw',180)

v2 = vehicle(scenario,'ClassID',1,'Position',[0 10 0],'Velocity',[0 -4 0],'Yaw',-90)

若要可视化方案，请对其调用绘图函数：

plot(scenario);
set(gcf,'Name','Scenario Plot')
xlim([-20 20]);
ylim([-20 20]);

创建场景中的所有Actor后，可以通过检查每个Actor的位置、滚动、俯仰、偏航、速度和角度速度属性来检查场景坐标中所有Actor的姿势信息，也可以通过在场景中调用actorPoses函数以方便的结构获取所有这些角色：

ap = actorPoses(scenario)

ap = 

  2x1 struct array with fields:

    ActorID
    Position
    Velocity
    Roll
    Pitch
    Yaw
    AngularVelocity

要获取特定Actor在其自己的参考系中看到的所有其他对象（或目标）的姿势信息，可以在Actor本身上调用targetPos函数：

v2TargetPoses = targetPoses(v2)

v2TargetPoses = 

  struct with fields:

            ActorID: 1
            ClassID: 1
           Position: [10 6.0000 0]
           Velocity: [-4 -3.0000 0]
               Roll: 0
              Pitch: 0
                Yaw: -90.0000
    AngularVelocity: [0 0 0]

我们可以通过添加车辆的追逐图来定性地确认相对车辆位置。默认情况下，追逐图显示车辆后方固定距离的投影透视视图。

在这里，我们显示了第二辆车（红色）后面的视角。第二辆车看到的目标姿势显示，另一辆车的位置（蓝色）在第二辆车前方6米，在第二辆车左侧10米处。我们可以在追逐剧情中定性地看到这一点：

chasePlot(v2)
set(gcf,'Name','Chase Plot')

通常，在调用 advance 函数时，与驾驶场景关联的所有绘图都会在仿真过程中更新。如果手动更新另一个 actor 的位置属性，则可以调用 updatePlot 立即查看结果：

v1.Yaw = 135;
updatePlots(scenario);

将道路边界转换为参与者的参考系

驾驶方案还可用于检索方案中定义的道路边界。

在这里，我们利用以编程方式定义道路布局中描述的简单椭圆形轨道，该轨道覆盖了大约 200 米长、100 米宽的区域，其曲线的倾斜角为 9 度：

scenario = drivingScenario;
roadCenters = ...
    [  0  40  49  50 100  50  49 40 -40 -49 -50 -100  -50  -49  -40    0
     -50 -50 -50 -50   0  50  50 50  50  50  50    0  -50  -50  -50  -50
       0   0 .45 .45 .45 .45 .45  0   0 .45 .45  .45  .45  .45    0    0]';
bankAngles = ...
    [  0   0   9   9   9   9   9  0   0   9   9    9    9    9    0    0];

road(scenario, roadCenters, bankAngles, 'lanes', lanespec(2));
plot(scenario);

要获取定义道路边界的线，请在驾驶方案中使用 roadBorders 函数。它返回一个包含道路边界的单元格数组（在上面的方案图中显示为黑色实线）。

rb = roadBoundaries(scenario)

rb =

  1x2 cell array

    {258x3 double}    {258x3 double}

在上面的示例中，有两个道路边界（外部边界和内部边界）。您可以按如下方式自行绘制它们：

figure

outerBoundary = rb{1};
innerBoundary = rb{2};

plot3(innerBoundary(:,1),innerBoundary(:,2),innerBoundary(:,3),'r', ...
      outerBoundary(:,1),outerBoundary(:,2),outerBoundary(:,3),'g')
axis equal

您可以使用Actor上的roadBorders函数来获取Actor坐标中的道路边界。为此，只需将 actor 作为第一个参数传递，而不是场景。

要看到这一点，添加一个“自我车辆”并将其放在轨道上：

egoCar = vehicle(scenario,'ClassID',1,'Position',[80 -40 0.45],'Yaw',30);

接下来，在车辆上调用 roadBorders 函数并像以前一样绘制它。它将相对于车辆的坐标进行渲染：

figure

rb = roadBoundaries(egoCar)
outerBoundary = rb{1};
innerBoundary = rb{2};

plot3(innerBoundary(:,1),innerBoundary(:,2),innerBoundary(:,3),'r', ...
      outerBoundary(:,1),outerBoundary(:,2),outerBoundary(:,3),'g')
axis equal

rb =

  1x2 cell array

    {258x3 double}    {258x3 double}

指定参与者轨迹

您可以沿着预定义的三维路径定位和绘制任何特定角色。

以下是两辆车的示例，它们分别以 30 m/s 和 50 m/s 的速度在赛道上行驶，每辆车都在各自的车道上。我们通过设置偏移位置将偏移位置设置为半条车道宽度（2.7 米）来将汽车从道路中心偏移，对于轨道的倾斜角度部分，每侧的垂直高度为一半：

chasePlot(egoCar);
fastCar = vehicle(scenario,'ClassID',1);

d = 2.7/2;
h = .45/2;
roadOffset = [ 0  0  0  0  d  0  0  0  0  0  0 -d  0  0  0  0
              -d -d -d -d  0  d  d  d  d  d  d  0 -d -d -d -d
               0  0  h  h  h  h  h  0  0  h  h  h  h  h  0  0]';

rWayPoints = roadCenters + roadOffset;
lWayPoints = roadCenters - roadOffset;

% loop around the track four times
rWayPoints = [repmat(rWayPoints(1:end-1,:),5,1); rWayPoints(1,:)];
lWayPoints = [repmat(lWayPoints(1:end-1,:),5,1); lWayPoints(1,:)];

smoothTrajectory(egoCar,rWayPoints(:,:), 30);
smoothTrajectory(fastCar,lWayPoints(:,:), 50);

推进仿真

遵循轨迹的Actor通过呼叫驾驶场景的前进来更新。当调用前进时，每个遵循轨迹的Actor都会向前移动，相应的情节也会更新。只有定义了轨迹的参与者才会真正更新。这样您就可以在模拟运行时提供自己的逻辑。

方案中的 SampleTime 属性控制更新之间的时间间隔。默认情况下为 10 毫秒，但您可以使用任意分辨率指定它：

scenario.SampleTime = 0.02

scenario = 

  drivingScenario with properties:

        SampleTime: 0.0200
          StopTime: Inf
    SimulationTime: 0
         IsRunning: 1
            Actors: [1x2 driving.scenario.Vehicle]
          Barriers: [0x0 driving.scenario.Barrier]
       ParkingLots: [0x0 driving.scenario.ParkingLot]

您可以通过在 while 循环的条件中调用 advance 并在循环正文中放置命令来检查或修改方案来运行模拟。

当任何车辆的轨迹完成或达到可选的停止时间时，while 循环将自动终止。

scenario.StopTime = 4;
while advance(scenario)
  pause(0.001)
end

录制场景

为了方便，当事先知道所有Actor的轨迹时，您可以调用场景上的record函数来返回一个结构，其中包含每个Actor在每个时间步的姿势信息。

例如，您可以检查每个Actor在模拟的前100毫秒的姿势信息，并检查第五个记录的样本：

close all

scenario.StopTime = 0.100;
poseRecord = record(scenario)

r = poseRecord(5)
r.ActorPoses(1)
r.ActorPoses(2)

poseRecord = 

  1x5 struct array with fields:

    SimulationTime
    ActorPoses


r = 

  struct with fields:

    SimulationTime: 0.0800
        ActorPoses: [2x1 struct]


ans = 

  struct with fields:

            ActorID: 1
           Position: [2.4000 -51.3502 0]
           Velocity: [30.0000 -0.0038 0]
               Roll: 0
              Pitch: 0
                Yaw: -0.0073
    AngularVelocity: [0 0 -0.0823]


ans = 

  struct with fields:

            ActorID: 2
           Position: [4.0000 -48.6504 0]
           Velocity: [50.0000 -0.0105 0]
               Roll: 0
              Pitch: 0
                Yaw: -0.0120
    AngularVelocity: [0 0 -0.1235]

将多个视图与鸟瞰图合并

调试模拟时，您可能希望报告特定参与者的鸟瞰图中的“真实”数据，同时查看场景生成的图。为此，您可以首先创建一个图形，其中轴以自定义排列方式放置：

close all;
hFigure = figure;
hFigure.Position(3) = 900;

hPanel1 = uipanel(hFigure,'Units','Normalized','Position',[0 1/4 1/2 3/4],'Title','Scenario Plot');
hPanel2 = uipanel(hFigure,'Units','Normalized','Position',[0 0 1/2 1/4],'Title','Chase Plot');
hPanel3 = uipanel(hFigure,'Units','Normalized','Position',[1/2 0 1/2 1],'Title','Bird''s-Eye Plot');

hAxes1 = axes('Parent',hPanel1);
hAxes2 = axes('Parent',hPanel2);
hAxes3 = axes('Parent',hPanel3);

定义轴后，您可以在创建图时通过 Parent 属性指定它们：

% assign scenario plot to first axes and add indicators for ActorIDs 1 and 2
plot(scenario, 'Parent', hAxes1,'ActorIndicators',[1 2]);

% assign chase plot to second axes
chasePlot(egoCar, 'Parent', hAxes2);

% assign bird's-eye plot to third axes
egoCarBEP = birdsEyePlot('Parent',hAxes3,'XLimits',[-200 200],'YLimits',[-240 240]);
fastTrackPlotter = trackPlotter(egoCarBEP,'MarkerEdgeColor','red','DisplayName','target','VelocityScaling',.5);
egoTrackPlotter = trackPlotter(egoCarBEP,'MarkerEdgeColor','blue','DisplayName','ego','VelocityScaling',.5);
egoLanePlotter = laneBoundaryPlotter(egoCarBEP);
plotTrack(egoTrackPlotter, [0 0]);
egoOutlinePlotter = outlinePlotter(egoCarBEP);

您现在可以重新启动模拟并运行它完成，这次通过 targetPose 提取目标汽车的位置信息并将其显示在鸟瞰图中。同样，你也可以直接从自我车辆中调用roadBorders和targetOutline 来提取道路边界和Actor的轮廓。鸟瞰图能够直接显示这些函数的结果：

restart(scenario)
scenario.StopTime = Inf;

while advance(scenario)
    t = targetPoses(egoCar);
    plotTrack(fastTrackPlotter, t.Position, t.Velocity);
    rbs = roadBoundaries(egoCar);
    plotLaneBoundary(egoLanePlotter, rbs);
    [position, yaw, length, width, originOffset, color] = targetOutlines(egoCar);
    plotOutline(egoOutlinePlotter, position, yaw, length, width, 'OriginOffset', originOffset, 'Color', color);
end
