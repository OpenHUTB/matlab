plotMesh

在鸟瞰图上显示对象的网格

collapse all in page

语法

plotMesh(mPlotter,vertices,faces)
plotMesh(mPlotter,vertices,faces,'Color',colors)

描述

plotMesh(mPlotter,vertices,faces) 在鸟瞰图上显示由指定的顶点和面组成的 meshes。要获取驾驶场景中对象的网格顶点和面，请使用 targetMeshes 函数。网格绘图器 mPlotter与 birdsEyePlot 对象相关联，并配置网格的显示。

鸟瞰图根据 Axes 对象的默认颜色顺序为每个参与者分配不同的颜色。有关更多详细信息，请参阅 Axes 对象的 ColorOrder 属性。

要移除与网格绘图器 mPlotter 关联的所有网格，调用 clearData 函数并将 mPlotter 指定为输入参数。

example

plotMesh(mPlotter,vertices,faces,'Color',colors) 指定网格的颜色。

例子

collapse all

在驾驶场景中显示参与者网格

Open Script

通过使用参与者的网格表示而不是其长方体表示，在驾驶场景中显示参与者。

创建一个驾驶场景，并向场景中添加一条25米的直路。

scenario = drivingScenario;
roadcenters = [0 0 0; 25 0 0];
road(scenario,roadcenters);

向场景中添加一个行人和一辆车辆。使用预构建的网格来指定参与者的网格尺寸。

  • 将行人的网格指定为 driving.scenario.pedestrianMesh 对象。

  • 将车辆的网格指定为 driving.scenario.carMesh 对象。

p = actor(scenario,'ClassID',4, ...
            'Length',0.2,'Width',0.4, ...
            'Height',1.7,'Mesh',driving.scenario.pedestrianMesh);

v = vehicle(scenario,'ClassID',1, ...
            'Mesh',driving.scenario.carMesh);

为行人和车辆添加轨迹。

  • 指定行人以每秒1米的速度穿过道路。

  • 指定车辆以每秒10米的速度沿着道路行驶。

waypointsP = [15 -3 0; 15 3 0];
speedP = 1;
smoothTrajectory(p,waypointsP,speedP);

wayPointsV = [v.RearOverhang 0 0; (25 - v.Length + v.RearOverhang) 0 0];
speedV = 10;
smoothTrajectory(v,wayPointsV,speedV)

为车辆添加一个以车辆为中心的视角的情节。打开网格的显示。

chasePlot(v,'Meshes','on')

创建一个鸟瞰图情节，用于显示网格。同时创建一个网格绘制器和车道边界绘制器。然后运行仿真循环。

  1. 获取车辆所在道路的道路边界。

  2. 获取参与者网格的网格顶点、面和颜色，相对于车辆的位置。

  3. 在鸟瞰图上绘制道路边界和参与者网格。

  4. 暂停场景，以便给绘图留出更新时间。追踪情节在推进场景时会更新。

bep = birdsEyePlot('XLim',[-25 25],'YLim',[-10 10]);
mPlotter = meshPlotter(bep);
lbPlotter = laneBoundaryPlotter(bep);
legend('off')

while advance(scenario)

   rb = roadBoundaries(v);

   [vertices,faces,colors] = targetMeshes(v);

   plotLaneBoundary(lbPlotter,rb)
   plotMesh(mPlotter,vertices,faces,'Color',colors)

   pause(0.01)
end

输入参数

collapse all

mPlotter — Mesh plotter
MeshPlotter object

网格绘制器，指定为 MeshPlotter 对象。该对象存储在 birdsEyePlot 对象的 Plotters 属性中，用于配置鸟瞰图中网格的显示。要创建此对象，请使用 meshPlotter 函数。

vertices — Mesh vertices of each actor
N-element cell array

每个参与者的网格顶点，指定为一个N元素的cell数组，其中N是参与者的数量。

vertices 中的每个元素必须是一个V×3的实值矩阵，其中包含一个参与者的顶点，其中：

  • V 是顶点的数量。

  • 每行定义了一个顶点的三维 (x,y,z) 位置。当您使用 targetMeshes 函数获取网格顶点时，顶点位置是相对于输入到该函数的演员的位置的。单位为米。

faces — Mesh faces of each actor
N-element cell array

每个参与者的网格面，以N个元素的单元数组形式指定，其中N是参与者的数量。

faces 中的每个元素必须是一个Fx3的整数值矩阵，其中包含一个参与者的面，其中：

  • F 是面的数量。

  • 每一行定义了构成面的顶点ID的三角形。顶点ID对应于 vertices 中的行号。

假设第 i 个元素的第一个 faces 具有以下顶点ID。

faces{i}(1,:)
ans =

     1     2     3

在 vertices 的第 i 个元素中，第1、2和3行包含构成该面的顶点的 (x, y, z) 位置。

vertices{i}(1:3,:)
ans =

    3.7000    0.9000    0.8574
    3.7000   -0.9000    0.8574
    3.7000   -0.9000    0.3149

colors — Color of mesh faces for each actor
N-by-3 matrix of RGB triplets

每个参与者的网格面的颜色，指定为一个Nx3的RGB三元组矩阵。N是参与者的数量，等于 vertices 和 faces 的元素数量。

第 i 行的 colors 是第 i 个元素的 faces 的RGB颜色值。该函数将相同的颜色应用于参与者的所有网格面。

RGB三元组是一个包含三个元素的行向量，这些元素指定颜色的红色、绿色和蓝色分量的强度。强度必须在[0, 1]的范围内。例如，[0.4 0.6 0.7]。

Meshes

In driving scenarios, a mesh is a triangle-based 3-D representation of an object. Mesh representations of objects are more detailed than the default cuboid (box-shaped) representations of objects. Meshes are useful for generating synthetic point cloud data from a driving scenario.
