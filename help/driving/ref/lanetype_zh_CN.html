laneType

创建道路车道类型对象

collapse all in page

语法

lt = laneType(type)
lt = laneType(type,Name,Value)

描述

example

lt = laneType(type) 返回具有属性 Type、Color和 Strength 的道路车道类型对象，用于定义道路的不同车道类型。

您可以使用此对象创建具有行车道、边界车道、限制车道、路肩车道和停车车道的驾驶场景。您还可以将该场景加载到 Driving Scenario Designer 应用程序中。

有关使用 laneType 函数与 drivingScenario 对象和 Driving Scenario Designer 应用程序的步骤详细信息，请参阅 More About。

example

lt = laneType(type,Name,Value) 通过使用一个或多个名称-值对设置输出`lane type`对象的属性。

例子

collapse all

在驾驶场景中添加不同车道类型的道路

Open Live Script

这个例子演示了如何定义车道类型，并为具有不同车道类型的四车道道路仿真一个驾驶场景。

创建一个具有默认属性值的驾驶车道对象。 

drivingLane = laneType('Driving')
drivingLane = 
  DrivingLaneType with properties:

        Type: Driving
       Color: [0.8000 0.8000 0.8000]
    Strength: 1

创建一个停车车道类型对象。指定颜色和强度属性值。

parkingLane = laneType('Parking','Color',[1 0 0],'Strength',0.1)
parkingLane = 
  ParkingLaneType with properties:

        Type: Parking
       Color: [1 0 0]
    Strength: 0.1000

通过连接驾驶车道类型对象和停车车道类型对象，创建一个包含三个元素的异构车道类型数组。该车道类型数组包含了一个四车道道路的车道类型。

lt = [parkingLane drivingLane drivingLane parkingLane];

为一个四车道道路创建车道规范。将车道类型数组添加到车道规范中。

ls = lanespec([2 2],'Type',lt);

创建一个驾驶场景对象。将具有车道规范 ls 的四车道道路添加到驾驶场景中。

scenario = drivingScenario;
roadCenters = [0 0 0;40 0 0];
road(scenario,roadCenters,'Lanes',ls)
ans = 
  Road with properties:

           Name: ""
         RoadID: 1
    RoadCenters: [2x3 double]
      RoadWidth: 14.5500
      BankAngle: [2x1 double]
        Heading: [2x1 double]

绘制场景。场景包含了两个停车车道和两个行驶车道的四车道道路。

plot(scenario)
legend('Driving Lane','Parking Lane')

仿真车辆在有多种车道类型的道路上行驶

Open Live Script

创建一个异构车道类型对象数组，用于定义四车道道路的行驶车道、路肩和边界车道类型。

lt = [laneType('Shoulder') laneType('Driving') laneType('Driving') laneType('Border','Color',[0.5 0 1],'Strength',0.1)];

显示车道类型对象数组。

lt
lt=1×4 object
  1x4 heterogeneous LaneType (ShoulderLaneType, DrivingLaneType, BorderLaneType) array with properties:

    Type
    Color
    Strength

检查属性值。

c = [{lt.Type}' {lt.Color}' {lt.Strength}'];
cell2table(c,'VariableNames',{'Type','Color','Strength'})
ans=4×3 table
      Type             Color            Strength
    ________    ____________________    ________

    Shoulder    0.59    0.59    0.59        1   
    Driving      0.8     0.8     0.8        1   
    Driving      0.8     0.8     0.8        1   
    Border       0.5       0       1      0.1

将车道类型对象数组作为输入传递给 lanespec 函数，然后为四车道道路创建一个车道规范对象。

lspec = lanespec([2 2],'Type',lt);

定义道路中心。

roadCenters = [0 0 0; 40 0 0];

要添加道路，请创建一个驾驶场景对象。

scenario = drivingScenario('StopTime',8);

在驾驶场景中添加具有指定道路中心和车道类型的道路。

road(scenario,roadCenters,'Lanes',lspec);

在场景中添加两辆车辆。将车辆定位在行驶车道上。

vehicle1 = vehicle(scenario,'ClassID',1,'Position',[5 2 0]);
vehicle2 = vehicle(scenario,'ClassID',1,'Position',[35 -2 0]);

使用航点定义车辆轨迹。设置车辆轨迹速度。

waypoints1 = [5 2;10 2;20 2;25 2;30 5;34 5.5];
smoothTrajectory(vehicle1,waypoints1,10)
waypoints2 = [35 -2;20 -2;10 -2;5 -2];
smoothTrajectory(vehicle2,waypoints2,5)

绘制场景。为了逐步推进仿真，可以在循环中调用 advance 函数。每0.01秒暂停一次，观察绘图中车辆的运动。第一辆车沿着行驶车道的轨迹行驶。它会漂移到路肩上进行紧急停车。

% Create a custom figure window and define an axes object
fig = figure;
movegui(fig,'center');
hView = uipanel(fig,'Position',[0 0 1 1],'Title','Scenario with Shoulder, Driving, and Border Lanes');
hPlt = axes(hView);

% Plot the generated driving scenario along with the waypoints.
plot(scenario,'Waypoints','on','Parent',hPlt);
while advance(scenario)
    pause(0.01)
end

输入参数

collapse all

type — 车道类型
'Driving' | 'Border' | 'Restricted' | 'Shoulder' | 'Parking'

Lane type, specified as 'Driving', 'Border', 'Restricted', 'Shoulder', or 'Parking'.

Lane Type Description
'Driving' 行驶车道
'Border' 道路边界的车道
'Restricted' 为高载客量车辆保留的车道
'Shoulder' 为紧急停车预留车道
'Parking' 车道旁边的车道，用于停放车辆

注意

车道类型的输入设置了输出车道类型对象的 Type 属性。

Data Types: char | string

名称-值参数

将可选的参数对指定为 Name1=Value1,...,NameN=ValueN 的形式，其中 Name 是参数名称，Value 是相应的值。名称-值参数必须出现在其他参数之后，但是参数对的顺序无关紧要。

在R2021a之前，使用逗号将每个名称和值分隔开，并用引号括住 Name 。

Example: laneType('Driving','Color','r')

Color — 车道颜色
RGB triplet | color name

车道颜色，指定为由'Color'和RGB三元组或颜色名称组成的逗号分隔的参数对。

将RGB三元组指定为包含该颜色的红、绿、蓝分量强度的三元素行向量。强度必须在[0,1]范围内，例如，[0.4 0.6 0.7]。该表列出了指定不同通道类型的默认颜色的RGB三元组值。

Lane Type RGB Triplet (Default values) Appearance
'Driving' [0.8 0.8 0.8]

'Border' [0.72 0.72 0.72]

'Restricted' [0.59 0.56 0.62]

'Shoulder' [0.59 0.59 0.59]

'Parking' [0.28 0.28 0.28]

或者，您可以通过名称指定一些常见的颜色。下表列出了命名颜色选项及其对应的RGB三元组值。

Color Name RGB Triplet Appearance
'red' [1 0 0]

'green' [0 1 0]

'blue' [0 0 1]

'cyan' [0 1 1]

'magenta' [1 0 1]

'yellow' [0.98 0.86 0.36]

'black' [0 0 0]

'white' [1 1 1]

注意

使用车道颜色名称-值对来设置输出车道类型对象的 Color 属性。

Data Types: single | double | int8 | int16 | int32 | int64 | uint8 | uint16 | uint32 | uint64 | char | string

Strength — 车道颜色的强度
1 (default) | real scalar in the range [0, 1]

车道颜色的强度是指通过'Strength'和一个取值范围为[0, 1]的实数标量组成的逗号分隔对。当值为 0 时，颜色会变得不饱和，车道颜色会呈现灰色。当值为 1 时，颜色会完全饱和，车道颜色为纯色。您可以调整强度值来修改饱和度的级别。

注意

使用车道颜色强度的名称-值对来设置车道类型对象的 Strength 属性。

Data Types: single | double | int8 | int16 | int32 | int64 | uint8 | uint16 | uint32 | uint64

输出参数

collapse all

lt — 车道类型
DrivingLaneType object | BorderLaneType object | RestrictedLaneType object | ShoulderLaneType object | ParkingLaneType object

车道类型，返回为

  • DrivingLaneType object

  • BorderLaneType object

  • RestrictedLaneType object

  • ShoulderLaneType object

  • ParkingLaneType object

返回的对象 lt 取决于输入 type 的值。

type lt
'Driving' DrivingLaneType object
'Border' BorderLaneType object
'Restricted' RestrictedLaneType object
'Shoulder' ShoulderLaneType object
'Parking' ParkingLaneType object

您可以通过连接这些不同的车道类型对象来创建异构 LaneType 数组。

collapse all

创建具有多个车道类型的道路的驾驶场景

您可以按照以下步骤将具有多个车道类型的道路添加到驾驶场景中：

  1. 创建一个空的 drivingScenario 对象。

  2. 使用 laneType 创建一个车道类型对象，定义道路上不同的车道类型。

  3. 将`lane type`对象作为输入，使用 lanespec 对象为道路定义车道规格。

  4. 将 lanespec 对象作为输入，使用 road 函数将具有指定车道类型的道路添加到驾驶场景中。

您可以使用 plot 函数可视化驾驶场景。

您还可以将包含不同车道类型的道路的驾驶场景导入到 Driving Scenario Designer 应用程序中。要将名为 scenario 的 drivingScenario 对象导入应用程序中，可以使用语法 drivingScenarioDesigner(scenario)。在场景中，您可以：

  • 添加或编辑道路中心。

  • 添加参与者并定义参与者轨迹。

  • 在自车上安装传感器，并仿真对参与者和车道边界的检测。

注意

编辑车道参数会将导入道路中的所有车道重置为车道类型为 'Driving'，并使用默认属性值。
