compositeLaneSpec

为道路创建多个车道规格

expand all in page

描述

compositeLaneSpec 对象将一组 lanespec 对象组合在一起，以创建具有不同车道规格的多个道路段的道路。

创建

要定义复合车道规格，请按照以下步骤进行操作：

  1. 创建一个 lanespec 对象的数组。lanespec 对象的数量定义了道路中的道路段数。每个 lanespec 对象包含一个道路段的车道规格。

  2. 创建一个复合车道规格对象 compositeLaneSpec，使用以下语法之一将车道规格组合在一起。默认情况下，函数假设每个道路段的范围相等。

  3. 若要为每个道路段设置不同的范围，请使用 compositeLaneSpec 对象的 SegmentRange 属性。

  4. 若要定义两个道路段之间的连接，请使用 compositeLanespec 对象的 Connector 属性。

  5. 使用 road 函数将 compositeLaneSpec 对象添加到驾驶场景中。

语法

clspec = compositeLaneSpec(lsArray)
clspec = compositeLaneSpec(lsArray,Name,Value)

描述

example

clspec = compositeLaneSpec(lsArray) 使用车道规格对象数组 lsArray 创建一个道路的复合车道规格。

例如，创建一个复合车道规格对象 compositeLaneSpec，将两个道路段的车道规格组合在一起。其中一个道路段有两个车道，另一个道路段有三个车道。

lsArray = [lanespec(2) lanespec(3)];
clspec = compositeLaneSpec(lsArray);
对于道路的车道和道路段的编号顺序取决于道路的方向。有关更多详细信息，请参阅 Draw Direction of Road and Numbering of Lanes 和 Composite Lane Specification。

example

clspec = compositeLaneSpec(lsArray,Name,Value) 使用一个或多个名称-值参数设置属性。例如，'SegmentRange',[0.6 0.4] 指定了两个道路段的归一化范围分别为 0.6 和 0.4。

属性

expand all

LaneSpecification — Lane specifications of road segments
1-by-N array of lanespec objects

该属性是只读的。

道路段的车道规格，以1xN的 lanespec 对象数组形式指定。N是车道规格的数量。要创建具有多个车道规格的道路，您必须指定至少两个 lanespec 对象。每个 lanespec 对象代表一个独立的道路段。因此，lanespec 对象的数量定义了道路中的道路段数量。

您必须在创建对象时使用 lsArray 输入参数来设置此属性。要创建车道规格对象，请使用 lanespec 函数。

Example: [lanespec(2) lanespec(3)] 定义了一个具有两个道路段的道路的车道规格。第一个道路段有两个车道，第二个道路段有三个车道。

SegmentRange — Range of each road segment
N-element numeric vector

每个道路段的范围，以具有标准化值的N元数值向量表示，范围为 (0, 1)。N是 LaneSpecification 属性中的 lanespec 对象数量。向量的元素之和必须等于 1。

每个道路段的默认范围值是 1/N。例如，如果 LaneSpecification 属性定义了两个道路段的车道规格，则每个道路段的默认范围值为 1/2，即 SegmentRange = [0.5 0.5]。

Example: [0.4 0.3 0.3]

Data Types: single | double

Connector — Road segment connectors
laneSpecConnector object | 1-by-M array of laneSpecConnector objects

道路段连接器，指定为 laneSpecConnector 对象或1xM的 laneSpecConnector 对象数组。M 是道路段连接器的数量。对于具有N个段的道路，M = N – 1。当指定为单个对象时，相同的规格适用于道路的所有M个连接器。

laneSpecConnector 对象指定了连接一对道路段的属性。您可以将这些属性作为名称-值参数指定给 laneSpecConnector 创建函数。

  • TaperShape — 将连接两个道路段的锥度形状指定为'Linear' 或 'None'。当您希望在道路段之间增加或减少车道时进行阶跃变化时，指定为 'None'。TaperShape 的默认值为 'Linear'。

  • TaperLength — 指定锥度的长度，单位为米。TaperLength 的默认值为 241 米或包含锥度的道路段长度的 75 ％中较小的值。

  • Position — 指定从哪一侧的道路添加或删除车道。您可以将连接器位置指定为 'Right', 'Left' 或 'Both'。

Example: [laneSpecConnector('Position','Left') laneSpecConnector('TaperLength',20)]

注意

  • 锥度是在配对中较低编号的道路段的一部分。有关对道路段进行编号的顺序的更多信息，请参阅 Composite Lane Specification。

  • 道路段连接器规格必须符合车道规格和段范围值。否则，该函数将使用有效值重置连接器规格。例如，如果您指定的 TaperLength 大于相应道路段的长度，该函数将使用相应道路段长度的 75 %来重置锥度长度。

例子

collapse all

从双向道路中去除右侧车道

Open Live Script

创建一个具有合并交通的驾驶场景。驾驶场景中的道路具有两个车道规格，并且交通在右侧车道结束时向左合并。

创建具有两个车道规格的道路

创建一个驾驶场景。指定道路中心以从底部到顶部的绘制方向。 

scenario = drivingScenario('StopTime',5);
roadCenters = [0 20; 100 20];

为一对双向道路段定义车道规格。第一个道路段有五条车道，第二个道路段有四条车道。请注意，第二个车道规格中的最右边车道被删除。

ls1 = lanespec([2 3],'Width',4);
ls2 = lanespec([2 2],'Width',4);
lsArray = [ls1 ls2];

创建一个复合车道规格对象，并将道路添加到驾驶场景中。复合车道规格对象确定了车道从 lsArray 输入参数中的哪个位置开始减少。该对象默认为线性锥度形状，锥度长度为第一个道路段长度的75%。

clspec = compositeLaneSpec(lsArray);
road(scenario,roadCenters,'Lanes',clspec);

绘制驾驶场景。场景按照道路的绘制方向，从底部到顶部呈现道路段。

figMark = figure;
set(figMark,'Position',[0 0 600 600])
hPlot = axes(figMark);
plot(scenario,'Parent',hPlot)
title('Two-way Road Dropping Rightmost Lane')

仿真车辆变道

在场景中添加一个自车。指定航点和一个恒定的速度值，以设置自车沿着中间前进车道的轨迹。

egoVehicle = vehicle(scenario,'ClassID',1,'Position',[2 16 0]);
waypoints = [2 16; 20 16; 95 16];
speed = 30;
smoothTrajectory(egoVehicle,waypoints,speed)

在场景中添加另一辆车辆。设置该车辆的轨迹，使其在最右车道行驶，然后在车道结束之前合并到左侧车道。

car = vehicle(scenario,'ClassID',1,'Position',[2 12 0]);
waypoints = [2 12; 12 12; 30 14; 42 16; 58 16; 85 16];
speed = 20;
smoothTrajectory(car,waypoints,speed)

创建一个自定义的图形窗口并绘制场景。

close all
figScene = figure;
set(figScene,'Position',[0 0 600 600])
hPanel = uipanel(figScene);
hPlot = axes(hPanel);
plot(scenario,'Waypoints','on','Parent',hPlot)
title('Merge Left as Right Lane Ends')

while advance(scenario)
    pause(0.01)
end

在单行道上添加车道

Open Live Script

创建一个具有多个车道规格的道路，并在单行道的左侧添加一个车道。

创建一个驾驶场景。指定 100 米直道路的道路中心，绘制方向从左到右。

scenario = drivingScenario;
roadCenters = [20 100; 20 0];

为两个单行道段定义一个车道规格的数组。第一个道路段有两个车道，第二个道路段有三个车道。

lsArray = [lanespec(2) lanespec(3)];

定义一个道路段连接器对象。为了在第二个道路段的左侧添加第三条车道，指定位置属性。指定一个小于第一个道路段长度的锥度长度。由于默认情况下，总道路长度为100米，将在指定的道路段之间平均分配，所以两个道路段的长度都为50米。

lc = laneSpecConnector('Position','Left','TaperLength',30);

创建一个复合车道规格对象。

clspec = compositeLaneSpec(lsArray,'Connector',lc);

在驾驶场景中添加一条道路并显示道路。场景按照道路的绘制方向，从左到右渲染道路段。

road(scenario,roadCenters,'Lanes',clspec);
plot(scenario)
title('Road with Multiple Lane Specifications')

在曲线上变化车道宽度

Open Live Script

创建一个空的驾驶场景。指定曲线道路的道路中心。 

scenario = drivingScenario;
roadCenters = [-20 22; 0 22; 18.8 15.8; 22 0; 22 -20];

为三个双向道路段定义车道规格。请注意，所有的道路段都有相同数量的车道。然而，第二个道路段的车道宽度较大（4.6米），以便在曲线上拓宽道路。其他两个道路段的车道宽度为默认值3.6米。

lsArray = [lanespec([1 1]) lanespec([1 1],'Width',4.6) lanespec([1 1])];

为每个道路段定义标准化长度。请注意，标准化长度的总和为 1，并且向量的长度与车道规格对象的数量相匹配。

range = [0.25 0.65 0.1];

创建一个道路段连接器对象。由于相同的规格适用于三个道路段的两个连接器，只需创建一个 laneSpecConnector 对象。由于既不增加也不减少车道，不需要定义道路段连接器的位置属性。

lc = laneSpecConnector('TaperLength',14);

创建一个复合车道规格对象，并将道路添加到驾驶场景中。 

clspec = compositeLaneSpec(lsArray,'Connector',lc,'SegmentRange',range);
road(scenario,roadCenters,'Lanes',clspec);

绘制驾驶场景。

plot(scenario,'RoadCenters','on')
title('Lane Widening Along a Curve')

限制

  • 在从一个道路段过渡到另一个道路段时，车道标线的间距是不一致的。

绘制道路的行驶方向和车道编号

通过使用 road 函数来创建道路，需要将道路中心点指定为矩阵输入。该函数创建一条有向线路，沿着道路中心点遍历，从矩阵的第一行坐标开始，到矩阵的最后一行坐标结束。矩阵的前两行坐标指定了道路的绘制方向。这些坐标对应于连续的前两个道路中心点。绘制方向是道路在场景图中呈现的方向。

通过使用 Driving Scenario Designer 应用程序创建道路时，可以通过指定 Road Centers 参数或在 Scenario Canvas 上进行交互式绘制来实现。详细示例，请参阅 Create a Driving Scenario。在这种情况下，绘制方向是道路在 Scenario Canvas 上呈现的方向。

  • 对于绘制方向自上而下的道路，前两个连续道路中心点的x坐标之差是正数。

  • 对于绘制方向自下而上的道路，前两个连续道路中心点的x坐标之差是负数。

  • 对于绘制方向从左到右的道路，前两个连续道路中心点的y坐标之差是正数。

  • 对于绘制方向从右到左的道路，前两个连续道路中心点的y坐标之差是负数。

编号车道

车道必须从左到右进行编号，其中道路的左边缘是相对于道路的绘制方向定义的。对于单行道，默认情况下，道路的左边缘是一条实心黄线，表示道路在横向方向（与绘制方向垂直的方向）的结束。对于双行道，默认情况下，两侧边缘都用实心白线标记。

例如，这些图示展示了绘制方向自上而下的单行道和双行道中车道的编号方式。

Numbering Lanes in a One-Way Road Numbering Lanes in a Two-Way Road

对于单行道，将车道数量指定为正整数。如果将整数值设置为 3，则该道路有三条在同一方向行驶的车道。车道的编号从道路的左边缘开始。

分别用1, 2, 3 表示道路的第一、第二和第三条车道。

对于双向道路，可以将车道数量指定为由两个正整数组成的向量。如果将向量设置为[1 2]，则该道路有三条车道：两条向同一方向行驶的车道和一条向相反方向行驶的车道。由于绘制方向的原因，该道路有一条左车道和两条右车道。车道从道路的左边缘开始编号。

1L 表示道路上唯一的左侧车道。1R 和 2R 分别表示道路上的第一和第二右侧车道。

车道规格按照车道编号的顺序应用。

复合车道规格

复合车道规格由一个包含两个或更多车道规格的数组组成，用于描述单个道路。每个车道规格定义了一个道路段，即具有独立几何属性、规范化范围和渐变的道路部分。

每个道路段都是一个有向段，朝着最终的道路中心移动，第一个段从第一个道路中心开始，第二个段从第一个段结束的地方开始，依此类推。每个道路段的范围是一个规范化距离，指定了道路总长度的比例。当一个道路段增加或减少车道时，前一个段会沿着指定的距离逐渐变窄，以适应车道数量的变化。

当使用复合车道规格渲染道路时，道路段按照道路的绘制方向进行绘制。例如，考虑一个单行道，有两个道路段，每个道路段的默认规范化范围为0.5。第一个道路段包含四个车道，第二个道路段只包含两个车道。第一个段从四个车道逐渐变窄到两个车道，当它接近道路的中点，即第二个段的起点时，每一侧都减少一个车道。以下示意图展示了道路段的绘制方向以及渐变如何应用于道路，分别针对从左到右和从右到左的绘制方向。

有关车道的几何属性的详细信息，请参阅 Lane Specifications。

车道规格

这个图示展示了道路、车道和车道标线的组成部分和几何属性。

车道规格对象（lanespec）定义了道路车道。

  • NumLanes 属性指定车道的数量。在创建该对象时，您必须指定车道的数量。

  • Width 属性指定每条车道的宽度。

  • Marking 属性包含道路上每个车道标线的规格。Marking 是一个车道标线对象的数组，每个对象对应一条车道。要创建这些对象，请使用 laneMarking 函数。车道标线的规格包括：

    • Type — 车道标线的类型 (实线、虚线等)

    • Width — 车道标线的宽度

    • Color — 车道标线的颜色

    • Strength — 车道标线颜色的饱和度值

    • Length — 对于虚线车道，每个虚线的长度

    • Space — 对于虚线车道，虚线之间的间距

    • SegmentRange — 对于复合车道标线，每个标记段的归一化长度

  • Type 属性包含道路上每条车道的车道类型规格。Type 可以是一个均质的车道类型对象，也可以是一个异质的车道类型数组。

    • 均质的车道类型对象包含道路上所有车道的车道类型规格。

    • 异质的车道类型数组包含车道类型对象的数组，每个对象对应一条车道。

    要创建这些对象，请使用 laneType 函数。车道类型规格包括：

    • Type — 车道类型 (行驶车道、边界车道等)

    • Color — 车道颜色

    • Strength — 车道颜色的强度
