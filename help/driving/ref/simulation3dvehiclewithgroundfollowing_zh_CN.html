Simulation 3D Vehicle with Ground Following

在三维环境中实现跟随地面的车辆

expand all in page
  • Library:

  • Automated Driving Toolbox / Simulation 3D

    Vehicle Dynamics Blockset / Vehicle Scenarios / Sim3D / Sim3D Vehicle / Components

描述

Simulation 3D Vehicle with Ground Following 模块实现在三维仿真环境中实现了四轮车辆。该环境使用虚幻引擎®进行渲染。 该模块使用车辆的输入 (X, Y) 位置和偏航角来调整车辆的仰角、滚动角和俯仰角，使其跟随地面地形。该模块车辆速度和航向，并调整每个车轮的转向角度和旋转。将此模块用于自动驾驶应用。

要使用此模块，请确保模型中存在“Simulation 3D Scene Configuration”模块。如果将 Simulation 3D Vehicle with Ground Following 模块的采样时间 Sample time 参数设置为 -1，则该模块将继承在 Simulation 3D Scene Configuration 模块中指定的采样时间。

该模块的输入使用 SAE J670 [1] 和 ISO 8855 [2] 中定义的车辆 Z-轴向上右手(right-handed, RH) 笛卡尔坐标系。坐标系是惯性坐标系，最初与车辆几何中心对齐：

  • X-轴沿着车辆的纵向轴线并指向前方。

  • Y-轴沿着车辆的横轴并指向左侧。

  • Z-轴指向上方。

当从Z轴、 Y 轴和 X 轴的正方向看时，Z 轴、Y轴和X轴 的偏航角、俯仰角和滚动角分别在顺时针方向上为正。车辆放置在场景的世界坐标系中。有关更多详细信息，请参阅 Coordinate Systems for Unreal Engine Simulation in Automated Driving Toolbox。

笔记

Simulation 3D Vehicle with Ground Following 模块必须在 Simulation 3D Scene Configuration 模块之前执行。这样，Simulation 3D Vehicle with Ground Following 模块会在虚幻引擎三维可视化环境接收信号数据之前准备好信号数据。要检查模块执行顺序，请右键单击块并选择“属性”。在 “常规”选项卡上，确认以下“优先级”设置： 

  • Simulation 3D Scene Configuration — 0

  • Simulation 3D Vehicle with Ground Following — -1

有关执行顺序的更多信息，请参阅 虚幻引擎自动驾驶仿真的工作原理。

可以配置 Simulation 3D Vehicle with Ground Following 模块以 导入自定义网格 和 控制车辆灯光。

局限性

  • 鸟瞰视野 无法从 Simulation 3D Scene Configuration 模块中找到真实信号，例如道路、车道和参与者。

Ports

输入

expand all

X — 车辆纵向位置
标量

车辆沿场景 X-轴的纵向位置。X 位于惯性 Z-向上坐标系中。单位为米。

初始位置 [X, Y, Z] (米) 的 X 值必须与仿真开始时该端口的值匹配。

要沿整个车辆路径指定 X 端口的多个位置，首先在MATLAB®中定义 X 路点的时间序列。然后，使用 From Workspace (Simulink) 模块将这些路点提供给 X。要了解如何选择和指定路点，请参阅 为虚幻引擎仿真选择路点 示例。

Y — 车辆横向位置
scalar

车辆沿场景 Y-轴的横向位置。Y 位于惯性 Z-向上坐标系中。单位为米。

初始位置 Initial position [X, Y, Z] (m) 参数的 Y 值必须与仿真开始时该端口的值匹配。

要沿整个车辆路径指定端口 Y 的多个位置，首先在 MATLAB 中定义 Y 路点的时间序列。然后，使用 From Workspace (Simulink) 模块将这些路点提供给 Y。要了解如何选择和指定路点，请参阅 Select Waypoints for Unreal Engine Simulation 示例。

Yaw — 车辆的偏航方向角
标量

车辆沿场景 Z-轴的偏航方向角。偏航角 Yaw 位于 Z-向上坐标系中。单位为度。

初始旋转 Initial rotation [Roll, Pitch, Yaw] (deg) 参数的偏航值必须与仿真开始时该端口的值匹配。

要沿整个车辆路径在偏航端口指定多个方位角 Yaw 首先在 MATLAB 中定义偏航航路点的时间序列。然后，使用 From Workspace (Simulink) 模块将这些路径点提供给。要了解如何选择和指定路点，请参阅 Select Waypoints for Unreal Engine Simulation 示例。

Light controls — 车灯的打开或关闭
1×6 向量

光控制输入信号，指定为 1×6 布尔向量. 。矢量的每个元素都会打开或关闭特定的车灯，如此表中所示。值为 1 则灯亮；值为 0 则关闭灯

依赖

要创建此端口，请在灯光控制（Light Controls）选项卡上选择启用灯光控制（Enable light controls）。

数据类型: Boolean

输出

expand all

Location — 车辆位置
1×3 的实值向量

车辆在场景中的位置 (X, Y, Z) ，以实值 1×3 向量形式返回。该位置基于车辆原点，该原点位于地面上车辆的几何中心。  位置值 Location 位于惯性 Z-向上世界坐标系中。单位为米。

依赖

要启用此端口，请在 Ground Truth 选项卡上选择输出位置 Output location (m) 和 orientation (rad)。

数据类型: double

Orientation — 车辆的朝向
1×3 实值向量

车辆分别围绕场景的 Z-轴、Y-轴、和 X-轴的偏航、俯仰和滚转方向角，返回为 1×3 的实值向量形式返回。该方向基于车辆原点，该原点位于地面上车辆的几何中心。朝向 Orientation 值位于惯性 Z-向上坐标系中。单位为弧度。

依赖

要启用此端口，请在 Ground Truth 选项卡上选择输出位置 Output location (m) 和 orientation (rad)。

数据类型: double

Initial position [X, Y, Z] (m) — 初始的车辆位置
[0, 0, 0] (默认) | 1×3 的实值向量

Initial rotation [Roll, Pitch, Yaw] (deg) — 车辆旋转的初始角度
[0, 0, 0] (默认) | 1×3 的实值向量

Name — 车辆名称
SimulinkVehicle1 (默认) | 车辆名称

车辆名称。默认情况下，在模型中使用该模块时，该模块会将 Name 参数设置为 SimulinkVehicleX. X 的值取决于模型中 Simulation 3D Vehicle with Ground Following 的数量 。

车辆名称显示为与车辆同一模型内任何自动驾驶工具箱™ 三维Simulation the Parent name 传感器块的父级名称 Parent name 参数中的选项。通过父名称参数，可以选择要安装传感器的车辆。

Sample time — 采样时间
-1 (默认) | 正标量

采样时间 Ts, 以秒为单位。图形帧速率是采样时间的倒数。

如果将采样时间设置为 -1，则该模块将使用在 Simulation 3D Scene Configuration 模块中指定的采样时间。

导入自定义网格

要导入自定义网格来定义自定义车辆，请按照下列步骤操作：

  1. 安装适用于虚幻引擎 4 项目的自动驾驶工具箱接口支持包。请参阅 Install Support Package for Customizing Scenes.

  2. 在模块参数 Parameters 选项卡上，将类型 Type 设置为 Custom。

  3. 在 Path to custom mesh 字段中，输入虚幻引擎项目中车辆网格物体的路径。例如，输入 /MathWorksSimulation/Vehicles/Muscle/Meshes/SK_MuscleCar.SK_MuscleCar。

    要创建自定义车辆网格体，请参阅 为虚幻编辑器准备自定义车辆网格。

  4. 使用自定义网格中的车辆尺寸在相应的块参数字段中输入尺寸。

Control Vehicle Lights

To control the lights of vehicles in a scene, follow these steps:

  1. Install the Automated Driving Toolbox Interface for Unreal Engine 4 Projects support package. See Install Support Package for Customizing Scenes.

  2. On the block Light Controls tab, select Enable light controls.

  3. Use the enabled parameters to specify the vehicle light for:

    • Headlights

    • Brake lights

    • Reverse lights

    • Turn signal lights

  4. Connect Boolean light control signals to the Signal lights input port.
