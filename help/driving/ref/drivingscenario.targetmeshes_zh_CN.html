targetMeshes

相对于特定参与者的网格顶点和面

collapse all in page

语法

[vertices,faces] = targetMeshes(ac)
[vertices,faces,colors] = targetMeshes(ac)

描述

[vertices,faces] = targetMeshes(ac) 相对于指定的参与者 ac，返回驾驶场景中所有参与者的网格顶点和面。当使用 birdsEyePlot 对象显示 meshes 时，您可以使用输出的网格信息作为 plotMesh 函数的输入。

example

[vertices,faces,colors] = targetMeshes(ac) 还返回每个参与者的网格面的颜色。

例子

collapse all

在驾驶场景中显示参与者的网格表示

Open Script

使用参与者的网格表示而不是长方体表示，在驾驶场景中显示参与者。

创建一个驾驶场景，并向场景中添加一条长度为25米的直路。

scenario = drivingScenario;
roadcenters = [0 0 0; 25 0 0];
road(scenario,roadcenters);

向场景中添加一个行人和一辆车辆。使用预定义的网格指定参与者的网格尺寸。

  • 将行人的网格指定为 driving.scenario.pedestrianMesh 对象。

  • 将车辆的网格指定为 driving.scenario.carMesh 对象。

p = actor(scenario,'ClassID',4, ...
            'Length',0.2,'Width',0.4, ...
            'Height',1.7,'Mesh',driving.scenario.pedestrianMesh);

v = vehicle(scenario,'ClassID',1, ...
            'Mesh',driving.scenario.carMesh);

为行人和车辆添加轨迹。

  • 指定行人以每秒1米的速度横穿马路。

  • 指定车辆以每秒10米的速度沿着道路行驶。

waypointsP = [15 -3 0; 15 3 0];
speedP = 1;
smoothTrajectory(p,waypointsP,speedP);

wayPointsV = [v.RearOverhang 0 0; (25 - v.Length + v.RearOverhang) 0 0];
speedV = 10;
smoothTrajectory(v,wayPointsV,speedV)

为车辆添加一个自车视角的图形绘制。打开网格的显示。

chasePlot(v,'Meshes','on')

创建一个鸟瞰图，用于显示网格。同时创建一个网格绘制器和车道边界绘制器。然后运行仿真循环。

  1. 获取车辆所在道路的道路边界。

  2. 获取参与者网格的顶点、面和颜色，相对于车辆的位置。

  3. 在鸟瞰图上绘制道路边界和参与者网格。

  4. 暂停场景，以便给绘图留出时间更新。追踪图在每次推进场景时更新。

bep = birdsEyePlot('XLim',[-25 25],'YLim',[-10 10]);
mPlotter = meshPlotter(bep);
lbPlotter = laneBoundaryPlotter(bep);
legend('off')

while advance(scenario)

   rb = roadBoundaries(v);

   [vertices,faces,colors] = targetMeshes(v);

   plotLaneBoundary(lbPlotter,rb)
   plotMesh(mPlotter,vertices,faces,'Color',colors)

   pause(0.01)
end

输入参数

collapse all

ac — Actor
Actor object | Vehicle object

属于 drivingScenario 对象的参与者，可以指定为 Actor 和 Vehicle 对象。要创建这些对象，请分别使用 actor 和 vehicle 函数。

输出参数

collapse all

vertices — 每个参与者的网格顶点
N-element cell array

每个参与者的网格顶点，返回为一个包含N个元素的cell数组，其中N是参与者的数量。

vertices 中的每个元素必须是一个V×3的实值矩阵，其中包含一个参与者的顶点，其中：

  • V 是顶点的数量。

  • 每一行定义一个顶点的3D (x,y,z)位置。顶点位置是相对于输入参与者 ac 的位置的。单位为米。

faces — 每个参与者的网格面片
N-element cell array

每个参与者的网格面，返回为一个包含N个元素的cell数组，其中N是参与者的数量。

faces 中的每个元素必须是一个F×3的整数值矩阵，其中包含一个参与者的面，其中：

  • F是面的数量。

  • 每一行定义一个由顶点ID组成的三角形面。顶点ID对应于 vertices 中的行号。

假设 faces 的第 i 个元素的第一个面具有这些顶点ID。

faces{i}(1,:)
ans =

     1     2     3

在 vertices 的第 i 个元素中，第1、2和3行包含组成该面的顶点的 (x, y, z) 位置。

vertices{i}(1:3,:)
ans =

    3.7000    0.9000    0.8574
    3.7000   -0.9000    0.8574
    3.7000   -0.9000    0.3149

colors — 每个参与者的网格面片颜色
N-by-3 matrix of N RGB triplets

每个参与者的网格面的颜色，以N×3的RGB三元组矩阵形式返回。N是参与者的数量，等于 vertices 和 faces 中的元素数量。

第 i 行的 colors 是 faces 中第 i 个元素的面的RGB颜色值。该函数将相同的颜色应用于参与者的所有网格面。

RGB三元组是一个包含三个元素的行向量，其元素指定颜色的红色、绿色和蓝色分量的强度。强度必须在[0, 1]范围内。例如，[0.4 0.6 0.7]。

网格模型

在驾驶场景中，网格是一个基于三角形的三维对象表示。相比于默认的长方体（盒状）对象表示，网格表示更加详细。网格对于从驾驶场景生成合成点云数据非常有用。

这个表格显示了在驾驶场景中，车辆的长方体表示和网格表示之间的差异。
