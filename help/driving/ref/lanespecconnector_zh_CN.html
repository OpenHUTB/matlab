laneSpecConnector

定义道路分段连接器规格

expand all in page

描述

laneSpecConnector 对象定义了连接具有不同车道规格的两个路段的规范。有关创建具有多车道规格的道路的更多详细信息，请参阅 compositeLaneSpec。

Creation

语法

lc = laneSpecConnector
lc = laneSpecConnector(Name,Value)

描述

example

lc = laneSpecConnector 创建一个道路段连接器对象，以连接具有不同车道规格的两个道路段。指定道路段连接器对象作为 compositeLaneSpec 对象的输入，以创建具有多个车道规格的道路。

example

lc = laneSpecConnector(Name,Value) 使用一个或多个名称-值参数设置属性。例如，'TaperLength'，20 指定锥度长度为20米。有关锥度和路段几何特性的更多信息，请参见 Composite Lane Specification。

属性

expand all

TaperShape — 锥形形状
'Linear' (default) | 'None' | character vector | string scalar

连接两个路段的锥度形状，指定为字符向量或字符串标量，部分或完全匹配 ConnectorTaperShape 枚举成员名。在创建对象时，将此属性指定为必须部分或完全匹配这些枚举成员名之一的字符向量或字符串标量。

Enumeration Member Name Enumerated Value Description Example (Using Left-To-Right Road)
'Linear' 0 当前路段在添加或减少车道时以线性方式逐渐变窄或变宽。

'None' 1 在添加或减少车道时，当前路段不会逐渐变窄或变宽，而是会突然改变。

Example: 'TaperShape','None'

TaperLength — 锥形段的长度
real positive scalar

连接两个道路段的锥形段的长度，以实数正标量表示。单位为米。默认的锥形段长度为 241 米或包含锥形段的道路段长度的 75 ％中较小的值。

注意

  • 锥度长度必须小于相应路段长度。否则，该函数将其重置为对应路段长度的 75 %。

  • 当锥度形状设置为 'None'时，不要指定锥度长度。该函数将忽略指定的输入。

Example: 'TaperLength',20

Data Types: double

Position — 道路段连接器位置
'Right' (default) | 'Left' | 'Both' | character vector | string scalar

道路段连接器位置，指定为字符向量或字符串标量，部分或完全匹配 ConnectorPosition 枚举成员名。此属性指定要从中添加或删除车道的道路边缘。在创建对象时，将此属性指定为必须部分或完全匹配这些枚举成员名之一的字符向量或字符串标量。

Enumeration Member Name Enumerated Value Description Example (Using Left-To-Right Road)
'Right' 0 从道路的右侧增加或减少车道。

'Left' 1 从道路的左边增加或减少车道。

'Both' 2 在道路两侧增加或减少车道。

只有在连接具有不同车道数的两个单行道路段时才使用此属性。要从单行道的两侧添加或删除车道，道路段的车道数必须相差偶数。

当连接非单行道的道路段时，请不要指定此属性，因为 compositeLaneSpec 对象会忽略该属性。

  • 要连接双向道路段，compositeLaneSpec 对象根据相应的车道规格对象定义的车道数确定连接器的位置。例如，如果双向道路段的车道数分别为 [1 1] 和[2 1]，compositeLaneSpec 对象将应用 'Left' 位置。

  • 为了连接单向和双向道路片段，compositeLaneSpec 对象会从道路的左侧添加或删除左侧（向后）车道。当两个道路片段之间的右侧（向前）车道数量相匹配时，该对象应用 'Left' 位置。否则，连接器位置将设置为 'Both'。例如，如果两个道路片段的车道规格分别为 [1 2] 和 2，则compositeLaneSpec 对象将应用 'Left' 位置。相反，如果两个道路片段的车道规格分别为 [1 2] 和 1，则compositeLaneSpec 对象将将位置属性设置为 'Both'。

注意

  • 在驾驶场景中，将单向道路上的所有车道都视为右侧（向前）车道，这意味着交通流向与道路的绘制方向相同。有关道路的绘制方向和车道编号的更多信息，请参阅 Draw Direction of Road and Numbering of Lanes。

  • 当道路片段之间的车道数量或段宽发生变化时，TaperShape 和 TaperLength 属性生效。然而，Position 属性仅在道路片段之间添加或删除车道时生效。

Example: 'Position','Both'

例子

collapse all

在单行道上添加车道

Open Live Script

创建一个具有多个车道规格的道路，并在单行道的左侧添加一个车道。

创建一个驾驶场景。指定一个直线的、长度为100米的道路，绘制方向从左到右。

scenario = drivingScenario;
roadCenters = [20 100; 20 0];

定义一个包含两个单行道路段的车道规格数组。第一个路段有两个车道，第二个路段有三个车道。

lsArray = [lanespec(2) lanespec(3)];

定义一个道路段连接器对象。为了在第二个道路段的左侧添加第三条车道，指定位置属性。指定一个小于第一个道路段长度的锥度长度。由于默认情况下，总道路长度100米在指定的道路段之间均匀分配，因此两个道路段的长度都为50米。

lc = laneSpecConnector('Position','Left','TaperLength',30);

创建一个复合车道规格对象。

clspec = compositeLaneSpec(lsArray,'Connector',lc);

在驾驶场景中添加一条道路并显示出来。场景按照道路的绘制方向，从左到右渲染道路段。

road(scenario,roadCenters,'Lanes',clspec);
plot(scenario)
title('Road with Multiple Lane Specifications')

曲线上的车道宽度变化

Open Live Script

创建一个空的驾驶场景。指定一个曲线道路的道路中心。 

scenario = drivingScenario;
roadCenters = [-20 22; 0 22; 18.8 15.8; 22 0; 22 -20];

定义三个双行道路段的车道规格。请注意，所有的道路段都有相同数量的车道。然而，第二个道路段的车道宽度较大（4.6米），以在曲线处加宽道路。其他两个道路段的车道宽度为默认的3.6米。

lsArray = [lanespec([1 1]) lanespec([1 1],'Width',4.6) lanespec([1 1])];

为每个道路段定义归一化长度。请注意，归一化长度之和为 1，并且向量的长度与车道规格对象的数量相匹配。

range = [0.25 0.65 0.1];

创建一个道路段连接器对象。由于相同的规格适用于三个道路段的段连接器，只创建一个 laneSpecConnector 对象。由于您既不添加也不减少车道，不需要定义道路段连接器的位置属性。

lc = laneSpecConnector('TaperLength',14);

创建一个复合车道规格对象，并将道路添加到驾驶场景中。 

clspec = compositeLaneSpec(lsArray,'Connector',lc,'SegmentRange',range);
road(scenario,roadCenters,'Lanes',clspec);

绘制驾驶场景。

plot(scenario,'RoadCenters','on')
title('Lane Widening Along a Curve')

绘制道路的行驶方向和车道编号

要使用 road 函数创建道路，请将道路中心点指定为矩阵输入。该函数将创建一条有向线路，沿着道路中心点行驶，从矩阵的第一行坐标开始，到矩阵的最后一行坐标结束。矩阵的前两行坐标指定了道路的绘制方向。这些坐标对应于前两个连续的道路中心点。绘制方向是道路在场景图中呈现的方向。

要使用 Driving Scenario Designer 应用程序创建道路，您可以通过指定 Road Centers 参数或在 Scenario Canvas 上进行交互式绘制来实现。有关详细示例，请参阅 Create a Driving Scenario。在这种情况下，绘制方向是道路在 Scenario Canvas 中呈现的方向。

  • 对于一个从上到下的绘制方向的道路，前两个连续道路中心点的 x 坐标之差是正数。

  • 对于一个从下到上的绘制方向的道路，前两个连续道路中心点的 x 坐标之差是负数。

  • 对于一个从左到右的绘制方向的道路，前两个连续道路中心点的 y 坐标之差是正数。

  • 对于一个从右到左的绘制方向的道路，前两个连续道路中心点的 y 坐标之差是负数。

车道编号

车道必须从左到右进行编号，其中道路的左边缘是相对于道路的绘制方向定义的。对于单行道，默认情况下，道路的左边缘是一条实心黄色标线，它表示道路在横向方向（与绘制方向垂直的方向）上的结束点。对于双向道路，默认情况下，两侧都用实心白色线标记。

例如，这些图表显示了单向和双向道路上的车道编号，并从上到下绘制方向。

单行道上的车道编号 双向道路上的车道编号

请指定单行道的车道数量，以正整数表示。如果您将整数值设为 3，则表示该道路有三个车道，这些车道在同一方向上行驶。车道的编号从道路的左边缘开始。

1, 2, 3 分别表示道路的第一、第二和第三车道。

请使用一个包含两个正整数的向量来指定双行道的车道数量。如果您将向量设为 [1 2]，则表示该道路有三个车道：两个车道朝一个方向行驶，一个车道朝相反方向行驶。由于绘制方向，该道路有一个左车道和两个右车道。车道的编号从道路的左边缘开始。

1L 表示道路的唯一左车道。1R 和 2R 分别表示道路的第一和第二个右车道。

车道规格按照车道编号的顺序适用。

复合车道规格

复合车道规格由一个包含两个或多个车道规格的数组组成，用于描述单个道路。每个车道规格定义了一个道路段，该段具有独立的几何属性、归一化范围和锥度。

每个道路段都是一个定向段，向着最终的道路中心移动，第一个段从第一个道路中心开始，第二个段从第一个段结束的位置开始，依此类推。每个道路段的范围是一个归一化距离，指定了道路总长度的比例。当一个道路段增加或减少车道时，前一个段会沿着指定的距离逐渐变窄，以适应车道数量的变化。

当您使用复合车道规格渲染道路时，道路分段将按照道路的绘制方向渲染。例如，考虑一条有两个路段的单行道，每个路段的默认标准化范围为0.5。第一个路段包含四个车道，第二个路段只包含两个车道。第一段从四车道逐渐减少到两车道，每边减少一条车道，当它接近道路的中点时，这是第二段的起点。这些图表显示了道路段渲染的方向，以及从左到右和从右到左绘制方向的锥形如何应用于道路。

有关车道几何特性的信息，请参见 Lane Specifications。
