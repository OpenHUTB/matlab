驾驶场景设计器

设计驾驶场景、配置传感器并生成合成数据

expand all in page

描述

驾驶方案设计器应用能够设计合成用于测试自动驾驶系统的驾驶场景。

使用该应用程序，您可以：

  • 使用拖放界面创建道路和参与者模型。

  • 在智能车辆上配置安装视觉、雷达、激光雷达、INS和超声波传感器。你可以使用这些传感器生成参与者和车道边界检测结果、点云数据和惯性测量。

  • 加载代表欧洲新车评估计划的负载驾驶场景 （Euro NCAP） 测试协议 [1][2][3] 和其他预构建场景。

  • 将ASAM OpenDRIVE道路和车道导入驾驶场景。该应用程序支持 OpenDRIVE文件版本1.4和1.5，以及ASAM OpenDRIVE文件版本1.6。

  • 从OpenStreetMap导入道路数据，HERE高清实时地图®1，或 Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0） 2Web 服务进入驾驶场景。

    从Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) 服务导入数据需要Automated Driving Toolbox Importer for Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) Service.

  • 将驾驶场景中的道路网络导出为 ASAM OpenDRIVE 文件格式。该应用程序支持OpenDRIVE文件版本1.4和1.5，以及ASAM OpenDRIVE文件版本1.6。

  • 将驾驶场景中的道路网络、参与者和轨迹导出到 ASAM OpenSCENARIO 1.0 文件格式。

  • 将合成传感器检测导出到 MATLAB。

  • 生成场景和传感器的 MATLAB 代码，然后以编程方式修改场景并将其导回应用以进行进一步仿真。

  • 从场景和传感器生成 Simulink 模型，并使用生成的模型测试传感器融合或车辆控制算法。

要了解有关该应用程序的更多信息，请观看以下视频：

  • Sensor Simulation and Virtual Scene Design with the Driving Scenario Designer App, Part 1

  • Sensor Simulation and Virtual Scene Design with the Driving Scenario Designer App, Part 2

打开驾驶方案设计器应用

  • MATLAB 工具栏: 在“应用程序”选项卡上的“Automotive”下，单击应用程序图标。

  • MATLAB 命令提示符: 输入 .drivingScenarioDesigner

创建驾驶方案

创建车辆在弯道上行驶的驾驶场景道路，并将道路和车辆模型导出到 MATLAB 工作区。

 打开驾驶方案设计器应用。

drivingScenarioDesigner

创建弯曲的道路。在应用程序工具行程中，单击添加道路。单击画布底部，将道路路径延伸到画布中间，然后再次单击画布。将道路路径延伸到画布顶部，然后双击以创建道路。要使曲线更加复杂，请单击并拖动道路中心（开放圆圈），双击道路以添加更多道路中心，或双击道路中心表的标题 （°） 列中的条目以指定航向角作为道路中心点的约束。

向道路添加车道。在左侧窗格的“道路”选项卡上，展开“车道”部分。将车道数设置为 2。默认情况下，道路是单向的，两侧都有实心车道标记以指示路肩。

在道路的一端添加车辆。在应用工具行程中，选择添加Actor>汽车。然后单击道路以设置汽车的初始位置。

设置汽车的行驶轨迹。右键单击汽车，选择添加前进航点，然后添加汽车通过的航点。添加最后一个航点后，按 Enter 键。汽车沿第一个航点的方向自动旋转。

在航点之间通过时调整汽车的速度。在左窗格的航点、速度、等待时间和偏航表中，设置自我飞行器进入每个航点段时的速度 v （m/s）。增加直线段的汽车速度，降低曲线段的速度。例如，轨迹有六个航点，将 v （m/s） 像元设置为 30、20、15、15、20 和 30。

运行方案，并根据需要调整设置。然后单击“存储>道路与演员”，将道路和汽车模型存储到 MAT 文件

从场景生成传感器数据

从预构建的 Euro NCAP 驾驶场景生成激光雷达点云数据。

  • 有关应用中提供的预生成方案的更多详细信息，请参阅驾驶方案Prebuilt Driving Scenarios in Driving Scenario Designer. 
  • 有关可用的 Euro NCAP 方案的更多详细信息，请参阅驾驶方案Euro NCAP Driving Scenarios in Driving Scenario Designer.

加载与行人儿童发生碰撞的 Euro NCAP 自动紧急制动 （AEB） 场景。在碰撞时，撞击点发生在汽车宽度的 50% 处。

path = fullfile(matlabroot,'toolbox','shared','drivingscenario', ...
    'PrebuiltScenarios','EuroNCAP');
addpath(genpath(path)) % Add folder to path
drivingScenarioDesigner('AEB_PedestrianChild_Nearside_50width.mat')
rmpath(path) % Remove folder from path

将激光雷达传感器添加到自我车辆。首先单击添加激光雷达。然后，在传感器画布上，单击汽车车顶中心的预定义传感器位置。激光雷达传感器在预定义位置显示为黑色。汽车周围的灰色是传感器的覆盖区域。

运行方案。通过在画布和视图之间切换来检查方案的不同方面。您可以在传感器画布和方案画布之间切换，也可以在鸟瞰图和以自我为中心的视图之间切换。

在鸟瞰图和以自我为中心的视图中，Actor显示为网格体而不是长方体。要更改显示设置，请使用应用程序工具行程中的显示选项。

将传感器数据导出到 MATLAB 工作区。单击导出 >导出传感器数据，输入工作空间变量名称，然后单击确定。

导入编程驾驶场景和传感器

以编程方式创建驾驶场景、雷达传感器和相机传感器。然后将方案和传感器导入应用。

使用 drivingScenario 对象创建简单的驾驶方案。在这种情况下，自我车辆以每秒 50 米的恒定速度在 30 米的路段上直线行驶。对于自我车辆，指定 ClassID 属性 1。此值对应于应用类 ID 1，它引用类 Car 的参与者。有关应用如何定义类的更多详细信息，请参阅“Actor”参数选项卡中的“类参数说明”。

scenario = drivingScenario;
roadCenters = [0 0 0; 50 0 0];
road(scenario,roadCenters);

egoVehicle = vehicle(scenario,'ClassID',1,'Position',[5 0 0]);
waypoints = [5 0 0; 45 0 0];
speed = 30;
smoothTrajectory(egoVehicle,waypoints,speed)

使用 drivingRadarDataGenerator 对象创建雷达传感器，并通过使用 visionDetectionGenerator 对象创建相机传感器 。将两个传感器放在车辆原点，雷达朝向前，相机朝后

radar = drivingRadarDataGenerator('MountingLocation',[0 0 0]);
camera = visionDetectionGenerator('SensorLocation',[0 0],'Yaw',-180);

将场景、前置雷达传感器和后置摄像头传感器导入应用。

drivingScenarioDesigner(scenario,{radar,camera})

然后，可以运行方案并修改方案和传感器。若要生成新的驾驶场景、驾驶雷达数据生成器和视觉检测生成器对象，请在应用工具行程上，选择“导出>导出 MATLAB 函数”，然后运行生成的函数。

生成场景和传感器的同步模型

加载包含传感器的驾驶场景并生成 来自场景和传感器的模拟模型。

将预构建的驾驶场景加载到应用中。该方案包含两辆车穿过交叉路口。自我车辆向北行驶并包含一个摄像头传感器。该传感器配置为检测物体和车道。

path = fullfile(matlabroot,'toolbox','shared','drivingscenario','PrebuiltScenarios');
addpath(genpath(path)) % Add folder to path
drivingScenarioDesigner('EgoVehicleGoesStraight_VehicleFromLeftGoesStraight.mat')
rmpath(path) % Remove folder from path

生成场景和传感器的 Simulink 模型。在应用程序工具行程中，选择导出>导出 Simulink 模型。如果出现提示，请保存方案文件。

场景读取器块从场景文件中读取道路和参与者。要更新模型中的方案数据，请在应用程序中更新方案并保存文件。

视觉检测生成器模块重新创建应用程序中定义的相机传感器。要更新模型中的传感器，请在应用程序中更新传感器，选择导出>导出传感器 Simulink 模型，然后将新生成的传感器块复制到模型中。如果在更新传感器时更新了任何道路或参与者，请选择导出>导出 Simulink 模型。在这种情况下，场景读取器块准确读取参与者配置文件数据并将其传递给传感器。

指定用于 3D 模拟的车辆轨迹

创建具有车辆轨迹的方案，您可以 稍后在 Simulink 中重新创建，以便在 3D 环境中进行仿真。

打开一个预构建的方案，以重新创建通过 3D 环境提供的默认场景。在应用程序工具之旅中，选择“打开>预生成方案”>“Simulation3D”，然后选择一个方案。例如，选择 DoubleLaneChange.mat 方案。


指定车辆及其轨迹。

更新车辆的尺寸以匹配 3D 模拟环境中预定义的车辆类型。

  1. 在“演员”选项卡上，选择所需的“3D 显示类型”选项

  2. 在应用程序工具行程中，选择“3D 显示”>“使用 3D 模拟 Actor 尺寸”。在场景画布中，角色尺寸将更新以匹配 3D 模拟环境中角色的预定义尺寸。

预览稍后在 Simulink 中重新创建场景时的外观。在应用程序工具行程中，选择 3D 显示> 3D 显示中的视图模拟。3D 显示窗口打开后，单击运行。

根据需要修改车辆和轨迹。避免更改场景中预定义的道路网络或执行组件。否则，应用程序方案将与您稍后在 Simulink 中重新创建的方案不匹配。如果更改方案，3D 显示窗口将关闭。

修改完方案后，可以在 Simulink 模型中重新创建该方案，以便在 3D 仿真环境中使用。

drivingScenarioDesigner(scenarioFileName)

drivingScenarioDesigner（scenarioFileName）打开应用程序并将指定的方案MAT文件加载到应用程序中。此文件必须是从应用程序保存的方案文件。此文件可以包括场景中的所有道路、参与者和传感器。它还可以仅包括道路和参与者组件，或仅包括传感器组件。

如果场景文件不在当前文件夹中或不在 MATLAB 路径上的文件夹中，请指定完整路径名。例如：

drivingScenarioDesigner('C:\Desktop\myDrivingScenario.mat');

您还可以加载预构建的方案文件。在加载预构建场景之前，请将包含该场景的文件夹添加到 MATLAB 路径。

drivingScenarioDesigner(scenario)

drivingScenarioDesigner（scenario） 将指定的 drivingScenario 对象加载到应用程序中。此对象中参与者的 ClassID 属性必须对应于应用中的以下默认类 ID 参数值：

  • 1 — Car

  • 2 — Truck

  • 3 — Bicycle

  • 4 — Pedestrian

  • 5 — Jersey Barrier

  • 6 — Guardrail

在应用程序中创建执行组件时，具有这些类 ID 值的执行组件具有一组默认尺寸、雷达横截面模式和其他属性。摄像机和雷达传感器根据类 ID 值指定的参与者类型以不同的方式处理检测。

将 drivingScenario 对象导入应用时，应用的行为取决于该方案中参与者的 ClassID。

  • 如果执行组件的 ClassID 为 0，则应用将返回错误。在 drivingScenario 对象中，ClassID 为 0 保留给未知或未分配类的对象。应用无法识别或使用此值。为这些参与者分配一个应用程序类 ID 值，然后再次导入 driveScenario 对象。

  • 如果参与者的非零类 ID 与类 ID 值不对应，则应用将返回错误。更改执行组件的类 ID 或向应用添加新的执行组件类。在应用工具行程中，选择“添加新执行组件”>“新执行组件类”。

  • 如果某个执行组件的属性与其相应的类 ID 执行组件的属性明显不同，则应用将返回警告。警告中引用的 ActorID 属性对应于“参与者”选项卡顶部列表中某个参与者的 ID 值。ID 值位于参与者名称之前。若要解决此警告，请考虑更新执行组件属性或其 ClassID 值。或者，考虑向应用添加新的执行组件类。

演员和车辆位置和尺寸

在驾驶场景中，演员是一个长方体（盒子形）物体，具有特定的长度、宽度、 和身高。Actor还具有雷达横截面（RCS）模式，以dBsm为单位指定，您 可以通过设置角方位角和高程坐标进行优化。演员的位置是 定义为其底面的中心。这个中心点用作演员的旋转 中心，其与地面的接触点及其在局部坐标系中的原点。在 此坐标系：

  • X 轴从角色指向前方。

  • Y 轴指向演员的左侧。

  • Z 轴从地面向上指向。

横滚、俯仰和偏航分别在向前看 X、Y 和 Z 轴时顺时针正向。

车辆是在轮子上移动的演员。车辆有三个额外的属性来控制其前轴和后轴的位置。

  • 轴距 — 前桥和后轴之间的距离

  • 前悬 — 车辆前部与前轴之间的距离

  • 后悬 — 后轴与车辆后部之间的距离

与其他类型的参与者不同，车辆的位置由地面上位于其后轴中心下方的点定义。该点对应于车辆的自然旋转中心。与非车辆参与者一样，此点是车辆局部坐标系中的原点，其中：

  • X 轴从车辆指向前方。

  • Y 轴指向车辆左侧。

  • Z 轴从地面向上指向。

横滚、俯仰和偏航分别在向前看 X、Y 和 Z 轴时顺时针正向。

长方体车辆的原点（即位置）与 3D 模拟环境中车辆的原点不同。在 3D 仿真环境中，车辆原点位于地面上，位于车辆的几何中心。

对于非车辆演员，原点是相同的，并且位于演员几何中心的底部。

在 Simulink 中，要将车辆从长方体原点转换为 3D 模拟原点，请使用长方体到 3D 仿真模块。有关3D模拟坐标的更多详细信息，请参阅自动驾驶工具箱中的虚幻引擎模拟坐标系。

绘制道路方向和车道编号

要使用道路函数创建道路，请将道路中心指定为矩阵输入。该函数创建一条穿过道路中心的有向线，从矩阵第一行的坐标开始，到矩阵最后一行的坐标结束。矩阵前两行中的坐标指定道路的绘制方向。这些坐标对应于前两个连续的道路中心。绘制方向是道路在方案图中的渲染方向。

要使用驾驶方案设计器应用创建道路，您可以指定道路中心参数或在方案画布上以交互方式绘制。有关详细示例，请参阅创建驾驶方案。在这种情况下，绘制方向是道路在方案画布中渲染的方向。

  • 对于具有从上到下绘制方向的道路，前两个连续道路中心的 x 坐标之差为正。

  • 对于具有自下而上绘制方向的道路，前两个连续道路中心的 x 坐标之差为负。

  • 对于具有从左向右绘制方向的道路，前两个连续道路中心的 y 坐标之差为正。

  • 对于具有从右向左绘制方向的道路，前两个连续道路中心的 y 坐标之差为负。

复合车道标记

复合车道标记包括两个或多个标记段，这些标记段定义沿车道的多种标记类型。复合车道标记的几何属性包括每种标记类型的几何属性和标记段的归一化长度。

指定标记段在复合车道标记中出现的顺序取决于道路的绘制方向。每个标记路段都是具有起点的定向路段，并朝向最后一个道路中心移动。第一个标记段从第一个道路中心开始，并在指定长度内向最后一个道路中心移动。第二个标记段从第一个标记段的终点开始，向指定长度的最后一个道路中心移动。相同的过程适用于您为复合车道标记指定的每个标记段。您可以通过指定范围输入参数来设置每个标记段的规范化长度。

例如，考虑一条具有两条车道的单行道。从道路左边缘开始的第二个车道标记是标记类型为实心和虚线的复合车道标记。每种标记类型的标准化范围为 0.5。第一个标记段是实心标记，第二个标记段是虚线标记。这些图表显示了标记段应用于从左到右和从右到左绘制道路方向的顺序。

有关车道标记的几何属性的信息，请参阅车道规范。

复合车道规格

复合车道规范由单个道路的两个或多个车道规范的数组组成。每个车道规范定义一个路段，该路段是具有独立几何属性、归一化范围和锥度的道路部分。

每个路段都是向最终道路中心移动的定向路段，第一路段从第一个路中心开始，第二路段从第一个路段结束的地方开始，依此类推。每个路段的范围是一个归一化距离，它指定道路总长度的比例。当路段从前一段添加或删除车道时，前一段路段沿指定距离逐渐变细，以适应车道数的变化。

使用复合车道规范渲染道路时，路段将沿道路的绘制方向进行渲染。例如，考虑具有两个路段的单行道，每个路段的默认归一化范围为 0.5。第一路段包含四条车道，第二段仅包含两条车道。第一段从四车道逐渐变细到两条车道，当它接近道路的中点时，每侧下降一条车道，这是第二段的起点。这些图显示了从左到右和从右到左绘制方向的道路段渲染方向，以及锥度如何应用于道路。

有关车道几何属性的信息，请参见车道规范。

技巧

  • 导入地图数据时，您指定的地图区域和您选择的道路数量会直接影响应用程序性能。要提高性能，请指定最小的地图区域并选择创建驾驶方案所需的最少道路。

  • 你可以撤消（按 Ctrl+Z）和重做（按 Ctrl+Y）在方案和传感器画布上所做的更改。例如，您可以使用这些快捷方式删除最近放置的道路中心或重做雷达传感器的移动。有关更多快捷方式，请参阅驾驶方案设计器的键盘快捷方式和鼠标操作

  • 在包含许多Actor的场景中，为了跟踪自我车辆，您可以在车辆周围添加一个指示器。在应用工具行程中，选择显示>显示自我指示器。围绕自我车辆的圆圈突出显示了车辆在场景中的位置。此圆圈不是传感器覆盖区域。
