在 Simulink 中使用合成雷达和视觉数据的传感器融合

This example uses:

  • Automated Driving Toolbox
  • Simulink
Open Model

此示例展示了如何在 Simulink® 中使用 Automated Driving Toolbox™实现跟踪和传感器融合的合成数据模拟。它紧随 Sensor Fusion Using Synthetic Radar and Vision Data  MATLAB® 示例。

介绍

仿真合成雷达和视觉检测提供了创建罕见和潜在危险事件并用它们测试车辆算法的能力。此示例涵盖了 Simulink 中的整个合成数据工作流。

模型的设置和概述

在运行这个例子之前，使用 Driving Scenario Designer 应用程序创建了与 Sensor Fusion Using Synthetic Radar and Vision Data 中定义的相同场景。然后将该场景中的道路和参与者保存到名为 OpenLoop.mat 的场景文件中。

Scenario Reader 模块从保存的文件中读取参与者姿态数据。该模块将参与者的姿态从场景的世界坐标系转换为自车坐标系。参与者的姿态通过该模块生成的总线进行传输。

传感器模拟子系统使用参与者姿势，生成合成雷达和视觉检测。模拟检测在多对象跟踪器块的输入处连接，其输出是已确认的轨迹列表。最后，Bird's-Eye Scope 可视化参与者、视觉和雷达探测、确认的轨道和道路边界。以下部分介绍此模型的主要块。

模拟传感器检测

在此示例中，您模拟具有 6 个雷达传感器和 2 个视觉传感器的自我车辆，覆盖 360 度视野。传感器有一些重叠和一些覆盖差距。ego车辆在车辆的前部和后部都配备了远程雷达传感器和视觉传感器。车辆的每一侧都有两个短程雷达传感器，每个传感器覆盖90度。每侧一个传感器覆盖从车辆中间到后部。每侧的另一个传感器从车辆中间向前覆盖。

当您打开传感器模拟子系统时，您可以看到两个 Vision Detection Generator 模块，配置为从自车的前方和后方生成检测结果。视觉检测生成器的输出连接到一个 Detection Concatenation 模块。接下来，该子系统包含六个 Driving Radar Data Generator 模块，配置如上文所述。雷达数据生成器的输出被配置为报告每个目标生成的检测结果的聚类质心。

跟踪和传感器融合

来自视觉和雷达传感器的检测必须首先连接起来，形成 Multi-Object Tracker 模块的单个输入。串联是使用额外的检测串联块完成的。

多对象跟踪器块负责融合来自所有检测的数据并跟踪自我车辆周围的对象。多目标跟踪器的配置参数与相应的 MATLAB 示例 Sensor Fusion Using Synthetic Radar and Vision Data 中使用的参数相同。多对象跟踪器块的输出是已确认轨道的列表。

创建和传播总线

此示例中各个模块的输入和输出都是 Simulink.Bus (Simulink) 对象。为了简化模型的编译和总线的创建，所有的Vision Detection Generator、Driving Radar Data Generator、Multi-Object Tracker和Detection Concatenation模块都有一个属性，用于定义输出总线名称的来源。当设置为'Auto'时，总线会自动创建，并将其名称传递给将该总线作为输入的模块。当设置为 'Property' 时，您可以定义输出总线的名称。以下图片显示了检测总线、单个检测总线、轨迹总线和单个轨迹总线。

显示

Bird's-Eye Scope 是一个模型级可视化工具，您可以从 Simulation工具栏中打开它。在模拟选项卡下的 Review Results 中，点击 Bird's-Eye Scope。打开范围后，点击 Find Signals 来设置信号。然后运行模拟以显示参与者、视觉和雷达检测、轨迹和道路边界。以下图片显示了该示例的鸟瞰图范围。
