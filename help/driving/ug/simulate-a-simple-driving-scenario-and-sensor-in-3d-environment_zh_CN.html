在虚幻引擎环境中模拟简单的驾驶场景和传感器

此示例使用：

  • 自动驾驶工具箱
  • 计算机视觉工具箱
  • Simulink
打开实时脚本

自动驾驶工具箱™提供了用于在使用Epic Games®虚幻引擎®的模拟环境中可视化传感器的模块。此模型在预构建的场景中模拟简单的驾驶场景，并使用鱼眼摄像头传感器从场景中捕获数据。使用此模型了解配置和模拟场景、车辆和传感器的基础知识。有关虚幻引擎模拟环境的更多背景知识，请参阅 用于自动驾驶的虚幻引擎模拟。

addpath(genpath(fullfile(matlabroot,'examples','driving')))

模型概述

该模型由以下主要组件组成：

  • 场景 — 模拟 3D 场景配置块配置您在其中模拟的场景。

  • 车辆 — 两个带有地面跟随模块的模拟 3D 车辆配置场景中的车辆并指定其轨迹。

  • 非车辆演员 — 模拟 3D 行人 and 模拟 3D 自行车 块配置行人和骑自行车的人，并分别指定其轨迹。

  • 传感器 — 模拟 3D 鱼眼相机配置用于捕获模拟数据的鱼眼镜头相机的安装位置和参数。视频查看器模块可视化此传感器的模拟输出。

model = 'SimpleScenarioAndSensorModel3DSimulation';
open_system(model)

检查场景

在模拟 3D 场景配置块中，场景名称参数确定进行模拟的场景。此模型使用大型停车场场景，但您可以在多个预构建场景中进行选择。要浏览场景，可以打开与 3D 场景对应的 2D 图像。

data = load('sim3d_SpatialReferences.mat');
spatialRef = data.spatialReference.LargeParkingLot;
figure; imshow('sim3d_LargeParkingLot.jpg',spatialRef)
set(gca,'YDir','normal')

sceneName = 'LargeParkingLot';
[sceneImage, sceneRef] = helperGetSceneImage(sceneName);

hScene = figure;
helperShowSceneImage(sceneImage, sceneRef)
title(sceneName)

要了解如何探索其他场景，请参阅相应的场景参考页面。

这个块的场景视图参数决定了虚幻引擎窗口显示场景的视图。在此块中，场景视图设置为EgoVehicle，这是本场景中ego车辆(带有传感器的车辆)的名称。在模拟过程中，虚幻引擎窗口从ego车辆后面显示场景。你也可以改变其他车辆的场景视图。要从场景的根(场景原点)显示场景，选择root。

块的Weather选项卡控制太阳位置和场景天气。这个场景被设置在中午，阴天有小雨。

检查车辆

具有地面跟随块的模拟3D车辆在场景中建模。

  • Ego车辆块车辆包含鱼眼相机传感器。这辆车是一辆红色的掀背车。

  • 目标车辆模块是传感器从中捕获数据的车辆。这辆车被设计成绿色SUV。

在模拟过程中，两辆车在停车场直线行驶50米。目标车辆在自我车辆正前方10米处。

xlabel('X (m)')
ylabel('Y (m)')

xyOffset = 3;

x = 45 - xyOffset;
y = 0 - xyOffset;
w = 3;
h = 6;
distAhead = 10;
distTraveled = 50;

egoStart = rectangle('Position',[x y w h],'FaceColor','r');
targetStart = rectangle('Position',[x y+distAhead w h],'FaceColor','g');

egoEnd = rectangle('Position',[x y+distTraveled w h],'FaceColor','r');
targetEnd = rectangle('Position',[x y+distTraveled+distAhead w h],'FaceColor','g');


xTextOffset = 30;
startText = text(x-xTextOffset, y+distAhead, 'Starting Positions', ...
    'EdgeColor','black','BackgroundColor','White');

endText = text(x-xTextOffset, y+distTraveled+distAhead, 'Ending Positions', ...
    'EdgeColor','black','BackgroundColor','White');

X, Y和偏航输入端口控制这些飞行器的轨迹。X和Y是场景的世界坐标，单位是米。偏航是飞行器的方向角，以度为单位。

自我车辆从(45,0)到(45,50)的位置行驶，方向从原点逆时针90度。为了对这个位置建模，输入端口值如下:

  • X是45的常数。

  • Y是模拟时间的倍数。一个数字时钟块每0.1秒输出模拟时间，持续5秒，这是模拟的停止时间。然后将这些模拟时间乘以10，得到Y值为[0 1 2 3…]50]，或1米，最多50米。

  • 偏航是一个90的常量。

目标车辆与自我车辆具有相同的X和偏航值。目标车辆的Y值总是比自我车辆的Y值多10米。

在两种车辆中，初始位置[X, Y, Z] (m)和初始旋转[Roll, Pitch, Yaw] (deg)参数反映了仿真开始时车辆的初始[X, Y, Z]和[Yaw, Pitch, Roll]值。

为了创建更逼真的轨迹，您可以交互式地从场景中获取路点，并将这些路点指定为具有地面跟随块的模拟3D车辆的输入。请参阅虚幻引擎模拟的选择路径点。

两辆车也有可选的灯光控制输入端口启用。这个端口允许您指定一个逻辑向量，指定是否打开前灯、刹车灯、倒车灯或转向信号灯。两辆车在矢量的第二个位置都有一个1，这打开了它们的低光束前灯。有关启用和控制车辆灯的更多详细信息，请参阅模拟3D车辆与地面跟随块参考页面。

检查非车辆参与者

仿真3D行人模块和仿真3D自行车模块分别对行人和自行车进行建模。行人在(40,27.7)处静止，朝向移动的车辆。打开仿真3D行人块参数，更改行人的类型，缩放其大小，并更改名称。您可以指定行人的类型为男性、女性或儿童。

骑自行车的人从(50,0)处出发，跟随左侧两辆行驶的车辆直线行驶。打开Simulation 3D Bicyclist块参数，缩放其大小并更改名称。

检查传感器

仿真3D鱼眼相机模块对场景中使用的传感器进行建模。打开这个块并检查它的参数。

  • “安装”选项卡包含确定传感器安装位置的参数。鱼眼相机传感器安装在ego车辆车顶的中心。

  • 参数选项卡包含鱼眼相机的固有相机参数。这些参数都被设置为默认值，除了Mapping coefficients参数。在这个参数中，第二个系数从0减小到-0.0005来模拟镜头畸变。

  • Ground Truth选项卡包含一个参数，用于以米和弧度为单位输出传感器的位置和方向。在这个模型中，块输出这些值，这样您就可以看到它们在模拟过程中是如何变化的。

该块输出从模拟中捕获的图像。在模拟过程中，Video Viewer块显示这些图像。

模拟模型

模拟模型。当模拟开始时，可视化引擎可能需要几秒钟的时间来初始化，特别是当您第一次运行它时。AutoVrtlEnv窗口显示3D环境中的场景视图。

视频查看器块显示鱼眼相机的输出。

sim(model);

要在模拟过程中改变场景的视图，请使用数字键盘上的数字1-9。

要鸟瞰现场，请按0。

在对模型进行仿真后，尝试修改相机的固有参数，观察对仿真的影响。您还可以更改传感器块的类型。例如，尝试用3D模拟相机块替换3D模拟鱼眼相机。有关可用传感器块的详细信息，请参见为虚幻引擎仿真选择传感器。

rmpath(genpath(fullfile(matlabroot,'examples','driving')));
