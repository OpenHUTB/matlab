在鸟瞰图范围内可视化传感器数据和轨迹

Bird's-Eye Scope可可视化来自 Simulink 模型的信号，这些信号表示驾驶场景的各个方面。使用该范围，您可以分析：

  • 视觉、雷达和激光雷达传感器的传感器覆盖范围

  • 参与者和车道边界的传感器检测

  • 场景中移动对象的轨迹

这个例子展示了如何在示波器上显示这些信号并在仿真过程中分析这些信号。

打开模型和范围

打开包含用于传感器检测和跟踪的信号的模型。使用此模型 在Sensor Fusion Using Synthetic Radar and Vision Data in Simulink示例中。还要添加模型的文件夹到 MATLAB 搜索 路径。

addpath(genpath(fullfile(matlabroot,'examples','driving')))
open_system('SyntheticDataSimulinkExample')

从 Simulink 工具之旅中打开示波器。在Review Results下，单击 Bird's-Eye Scope 。

查找信号

首次打开Bird's-Eye Scope时，scope 画布为空白，并且不显示任何信号。要从打开的模型中查找示波器可以显示的信号， 在“范围工具”上，单击查找信号。范围更新 框图并自动查找模型中的信号。

左窗格列出了示波器找到的所有信号。这些信号被分组 基于它们在模型中的来源。

Signal Group Description Signal Sources
Ground Truth

在这个场景中，道路边界、车道标线和障碍物

您无法修改此组或其任何信号。

要检查大型道路网络，请使用世界坐标视图窗口。请查看Vehicle and World Coordinate Views。

  • Scenario Reader block

Actors

场景中的参与者，包括自我车辆

你不能修改此组或其任何信号或子组。

要查看远离自动驾驶车辆的角色，请使用世界坐标视图窗口。请参阅Vehicle and World Coordinate Views。

  • Scenario Reader block

  • Vision Detection Generator, Driving Radar Data Generator, and Lidar Point Cloud Generator blocks (仅查看角色的配置文件信息，例如角色的长度、宽度和高度。)

    • 如果参与者配置文件信息未设置或不一致 在块之间，作用域将执行组件配置文件设置为 每个块的默认参与者配置文件值。

    • 自我车辆的配置文件始终设置为 每个块的默认配置文件。

您可以修改此组中的信号。

你可以重命名或删除子组，但不能重命名或删除顶级传感器覆盖范围组。您还可以添加子组和移动信号子组之间。如果删除子组，其信号将移动到顶级传感器覆盖范围组。

从视觉、雷达和激光雷达传感器获得的检测结果，分为视觉、雷达、激光雷达和超声波子组

您可以修改此组中的信号。

你可以重命名或删除子组，但不能重命名或删除顶级检测组。您还可以添加子组和 在子组之间移动信号。如果删除子组，其信号将移动到顶级检测组。

您可以在此修改信号 群。

您可以重命名或删除子组，但不能重命名或删除顶级轨道组。您还可以向其添加子组对信号进行分组并将信号移动到其中。如果删除子组，其信号将移动 到顶级轨道组。

  • Multi-Object Tracker block

  • Tracker blocks in Sensor Fusion and Tracking Toolbox™

鸟瞰镜显示自我中的曲目 车辆坐标。任何其他坐标系中的轨迹将显示为 作用域中的偏移量。
Other Applicable Signals

示波器无法自动分组的信号，例如组合的信号 来自多个传感器的信息

您可以在此修改信号组，但不能添加子组。

此组中的信号不在模拟期间显示。

  • 组合或聚类信号的块（例如Detection Concatenation块）

  • Vehicle To World and World To Vehicle blocks

  • 创建包含演员姿势的非虚拟 Simulink 总线的任何块

    有关演员姿势的详细信息，创建时所需的姿势信息 这些总线，请参阅场景的Actors输出端口 读取器块。

  • 创建包含检测的非虚拟 Simulink 总线的任何块

    有关创建时所需的检测信息的详细信息 这些总线，请参阅视觉的Object Detections和Lane Detections输出端口 检测发电机块。

  • 创建包含磁道的非虚拟 Simulink 总线的任何块

    有关创建这些时所需的轨道信息的详细信息 总线，请参阅多对象跟踪器块的Confirmed Tracks 输出端口。

在模拟之前但在点击“查找信号”之后，范围画布会显示除非自动驾驶车辆以外的所有地面真实信号，以及所有传感器覆盖信号。非自动驾驶车辆和“检测”和“跟踪”下的信号只有在模拟模型后才会显示。在模拟过程中，其他适用信号不会显示。如果您希望范围显示特定的信号，请在模拟之前将它们移动到适当的组中。如果适当的组不存在，请创建一个。

运行仿真

通过在Bird's-Eye Scope 范围内点击Run来模拟模型。范围画布会显示检测和跟踪结果。要显示图例，请在范围工具栏上点击“Legend”。

在模拟过程中，您可以执行以下操作：

  • 检查检测、跟踪、传感器覆盖区域和自车行为。默认视图以车辆坐标显示模拟，并以自车为中心。要查看自车周围的更大区域，或者查看场景的其他部分，在范围工具栏上点击World Coordinates。世界坐标视图窗口会显示整个场景，并在自车周围画圈。这个圈不是传感器覆盖区域。要返回默认显示的任一窗口，请将鼠标指针悬停在窗口上方，并在右上角点击出现的主页按钮。有关这些视图的更多详细信息，请参阅Vehicle and World Coordinate Views。

  • 更新信号属性。要访问信号的属性，请首先从左窗格中选择信号。然后，在范围工具栏上点击“属性”。使用这些属性，您可以显示或隐藏传感器覆盖区域或检测结果。此外，为了突出显示特定的传感器覆盖区域，您可以更改它们的颜色或透明度。

  • 更新鸟瞰视图设置，例如更改车辆坐标视图窗口的坐标轴限制或更改信号名称的显示方式。在范围工具栏上点击 Settings。在模拟过程中，无法更改Track position selector 和 Track velocity selector的设置。有关更多详细信息，请参阅Settings > Vehicle Coordinates View部分的 Bird's-Eye Scope 。

在模拟结束后，您可以隐藏某些检测或跟踪以供下一次模拟使用。在左窗格中，在 Detections或 Tracks下，右键单击要隐藏的信号。然后，选择 Move to Other Applicable 将该信号移动到 Other Applicable Signals 组中。要隐藏传感器覆盖区域，选择左窗格中对应的信号，然后在“属性”选项卡上取消选中Show Sensor Coverage参数。在模拟过程中，无法隐藏 Ground Truth 信号。

组织信号组（可选）

为了进一步组织信号，您可以重命名信号组或将信号移动到新的组中。例如，您可以将Vision和Radar子组重命名为Front of Car和 Back of Car。然后，根据新的组名，根据需要拖动信号将其移动到相应的组中。当您将信号拖动到新的组中时，信号的颜色会更改为与其所属组分配的颜色相匹配。

您无法重命名或删除左窗格中的顶级组，但可以重命名或删除任何子组。如果删除一个子组，其信号将移动到顶级组中。

更新模型并重新运行仿真

在运行模拟之后，修改模型并检查这些更改对鸟瞰视图的可视化效果。例如，在模型的传感器模拟子系统中，打开两个“Vision Detection Generator”块。这些块分别具有传感器索引为1和2。在每个块的Measurements 选项卡上，将 Maximum detection range (m) 参数减小到50。为了查看传感器覆盖范围的变化，重新运行模拟。

当您修改块参数时，可以重新运行模拟而无需重新查找信号。如果添加或删除块、端口或信号线，则必须在重新运行模拟之前再次点击“查找信号”。

保存并关闭模型

当您保存并关闭模型时，Bird's-Eye Scope的设置也会被保存。

如果重新打开模型和Bird's-Eye Scope，范围画布最初为空白。点击 Run 以运行模拟并可视化保存的信号属性。例如，如果在上一步中减小了检测范围，范围画布将显示这个减小的范围。


如果向模型中添加新的信号，请点击 Update Signals 以查找新的信号，然后点击 Run 以可视化带有新信号的模型。

注意

如果在关闭模型之前没有对Bird's-Eye Scope进行图形更改，则在重新打开模型之前必须再次查找信号才能运行模拟。图形更改包括以下内容：

  • 将信号拖动到新组

  • 启用图例或世界坐标视图窗口

  • 更改轴限制

  • 更改演员、车道标记或传感器的视觉属性 覆盖区域

完成模型仿真后，从 MATLAB 搜索路径。

rmpath(genpath(fullfile(matlabroot,'examples','driving')))
