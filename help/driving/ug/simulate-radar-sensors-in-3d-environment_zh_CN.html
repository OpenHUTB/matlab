在虚幻引擎环境中模拟视觉和雷达传感器

此示例使用：

  • Automated Driving Toolbox
  • Simulink
Open Script

此示例展示了如何在Simulink®中使用Epic Games®的Unreal Engine®仿真环境实现跟踪和传感器融合的合成数据模拟。它紧密地遵循 Sensor Fusion Using Synthetic Radar and Vision Data in Simulink 示例。

介绍

Automated Driving Toolbox™提供了用于创建、仿真和可视化虚拟驾驶场景的工具。通过这些场景，您可以模拟罕见和潜在危险的事件，从场景中生成合成雷达和视觉检测，并使用这些合成检测来测试车辆算法。该示例涵盖了在Simulink中使用3D仿真环境进行整个合成数据工作流程的内容。

模型设置和概述

在运行此示例之前，通过以下步骤创建了场景中的道路、参与者和轨迹：

  1. 从 Define Road Layouts Programmatically 3D场景中提取道路的中心位置，使用 Select Waypoints for Unreal Engine Simulation 中介绍的技术。

  2. 在 Driving Scenario Designer 中创建一条道路，将提取的位置作为道路中心值。

  3. 在道路上定义多辆移动车辆，其轨迹与 Sensor Fusion Using Synthetic Radar and Vision Data in Simulink 中定义的场景类似。

  4. 从 Driving Scenario Designer 中保存场景文件 curved_road.mat，并使用Scenario Reader块将其加载到模型中。

  5. 从Scenario Reader块输出车辆的姿态，并将其转换为世界坐标。使用Cuboid to 3D Simulation块将其转换为3D车辆块所需的X、Y和偏航角。

模型中的 Simulation 3D Vehicle with Ground Following 块使用Scenario Reader块提供的参与者姿态，在仿真的每个时间步中定义自车、前车和其他车辆的位置。

模拟传感器检测

在这个示例中，您模拟了一个自车，它的前保险杠上有一个视觉传感器，以及覆盖全方位视野的六个雷达传感器。自车装备有前后两个远程雷达。车辆的每一侧都有两个短程雷达，每个雷达覆盖90度。每一侧的一个雷达从车辆中间向后覆盖，另一个雷达从车辆中间向前覆盖。

Ego Sensors子系统包含一个 Simulation 3D Vision Detection Generator 块和六个 Simulation 3D Probabilistic Radar 块，用于建模先前描述的传感器。雷达块的输出使用 Detection Concatenation 块进行连接。在顶层模型中，雷达输出与视觉输出连接在一起，形成一个要由多目标跟踪器块融合的检测流。

open_system('SimulateSensorsIn3DEnvironmentModel/Ego Sensors')

概率雷达不仅可以"看到"参与者的物理尺寸（例如长度、宽度和高度），还对参与者的电学尺寸敏感。参与者的电学尺寸被称为雷达散射截面 (RCS)。在仿真中，使用 Simulation 3D Probabilistic Radar Configuration 块定义了车辆的RCS模式。

使用该块来为仿真中的所有参与者定义雷达散射截面（RCS）模式。如果某个参与者没有指定RCS模式，将使用默认的RCS值。

跟踪和传感器融合

在将自动驾驶汽车的雷达套件生成的检测结果进行融合之前，会使用一个辅助的检测聚类块对其进行预处理。然后，使用 Multi-Object Tracker 块进行融合。多目标跟踪器的配置参数与Simulink示例中的 Sensor Fusion Using Synthetic Radar and Vision Data in Simulink 相同。多目标跟踪器块的输出是一组确认的轨迹。

显示工具

总结

在这个例子中，你学习了如何从3D场景中提取道路中心，以便在Driving Scenario Designer应用程序中使用。你还学习了如何从道路段落中导出车辆轨迹，以在Simulink中的3D仿真环境中使用。然后，你学习了如何在Unreal Engine环境中配置概率相机模型和多个概率雷达模型，以及如何使用多目标跟踪器融合围绕自车周围的多个传感器的检测。跟踪器生成的确认轨迹可以用于控制算法，如自适应巡航控制（ACC）或前向碰撞警告（FCW）。
