将驾驶场景导出到 ASAM 开放场景文件

ASAM OpenSCENARIO®是一种开放的文件格式，用于描述驾驶场景的动态内容。使用 Driving Scenario Designer 应用程序，您可以将驾驶场景中的道路网络、参与者和轨迹导出为 ASAM OpenSCENARIO 文件。

该应用程序支持将驾驶场景导出为 ASAM OpenSCENARIO 文件的1.0和1.1版本。

要以编程方式将驾驶场景导出为 ASAM OpenSCENARIO 文件，请使用 drivingScenario 对象的 export 对象函数。

加载场景文件

要打开 Driving Scenario Designer 应用程序，请在 MATLAB® 命令提示符下输入以下命令：

drivingScenarioDesigner

要加载现有的场景文件，请在应用程序工具栏上选择 Open > Scenario File。您选择的文件必须是具有.mat文件扩展名的有效驾驶场景会话文件。或者，您可以使用 Driving Scenario Designer 应用程序创建一个新的场景，然后将该场景导出为 ASAM OpenSCENARIO 文件。有关如何创建场景的信息，请参阅Create Driving Scenario Interactively and Generate Synthetic Sensor Data。

导航到并打开这个驾驶场景，其中 matlabroot 是您的MATLAB根文件夹。

matlabroot/examples/driving/data/CutInScenario.mat

Scenario Canvas窗格显示了场景，其中一个自车和一个目标车辆沿着预定义的轨迹在一条直路上行驶，目标车辆会切入自车车道。

这个场景包含以下 ASAM OpenSCENARIO 动作：

  • SpeedAction — 自我车辆以初始速度15 m/s行驶，然后在到达指定航点时暂时停止。然后，自我车辆以10 m/s的速度行驶，直到场景结束。

  • VisibilityAction — 目标车辆在3秒后进入场景。

  • FollowTrajectoryAction — 车辆明确地按照指定的航点轨迹行驶。

运行场景并观察两辆车的行为。

导出到 ASAM OpenSCENARIO

要将场景中的道路网络、参与者和轨迹导出为 ASAM OpenSCENARIO 文件，请在应用程序工具栏上，选择 Export > ASAM OpenSCENARIO File。

应用程序将打开 ASAM OpenSCENARIO 导出对话框。

在 OpenSCENARIO 选项卡上，指定以下选项：

  • File Path — 使用 .xosc 或 .xml 文件扩展名为输出 ASAM OpenSCENARIO 文件指定名称和位置。默认情况下，应用程序将文件保存在当前工作目录中。要指定不同的文件位置，请单击 Browse。如果已存在具有指定名称的文件，则应用程序将覆盖现有文件。

  • File Version — 将输出文件的 ASAM OpenSCENARIO 版本指定为 Version 1.0 或 Version 1.1。

    Default: Version 1.0

在 OpenDRIVE 选项卡上，使用以下选项指定关联的 ASAM OpenDRIVE® 文件的信息：

  • File Version — 将输出文件的 ASAM OpenDRIVE 版本指定为Version 1.4, Version 1.5 或 Version 1.6。

    Default: Version 1.4

  • Export Actors — 选择此参数将角色导出到ASAM OpenDRIVE文件中。

    Default: on

一旦您指定了文件信息，请单击 Export。应用程序将创建一个ASAM OpenSCENARIO文件，该文件描述了场景中角色的动态行为，以及其他数据文件。每个数据文件包含有关特定场景元素（如车辆或行人）的信息。每个数据文件的名称都具有前缀 filename_，其中 filename 是使用 File Path 选项指定的名称。

Data File Element of Scenario
filename_OpenDRIVE.xodr 场景中的道路网络和障碍
filename_VehicleCatalog.xosc 车辆的特性
filename_PedestrianCatalog.xosc 行人的属性

注意

该应用程序根据场景的内容导出数据文件。例如，如果场景不包含任何行人，则不会导出数据文件 filename_PedestrianCatalog.xosc。

ASAM OpenSCENARIO表示

导出的文件以ASAM OpenSCENARIO格式包含驾驶场景角色及其参数的表示。

Name, Class, and Actor Properties

该应用程序将每个参与者的 Name, Class和 Actor Properties 导出到 filename_VehicleCatalog 文件中的 Vehicle 元素中。

<Vehicle name="Car1" vehicleCategory="car">
  <BoundingBox>
    <Center x="1.35" y="0" z="0"/>
    <Dimensions height="1.4" length="4.7" width="1.8"/>
  </BoundingBox>
  <Performance maxAcceleration="5.36" maxDeceleration="6" maxSpeed="39"/>
  <Axles>
    <FrontAxle maxSteering="0.5" positionX="2.8" positionZ="0.1905" trackWidth="1.8" wheelDiameter="0.381"/>
    <RearAxle maxSteering="0.5" positionX="0" positionZ="0.1905" trackWidth="1.8" wheelDiameter="0.381"/>
  </Axles>
  <Properties/>
</Vehicle>

ASAM OpenSCENARIO车辆包括没有等效参与者特性的参数。该表显示了导出的各种类型参与者的这些ASAM OpenSCENARIO参数的默认值。

航点

该应用程序将参与者的航点导出到输出 ASAM OpenSCENARIO 文件的 Init 部分中 FollowTrajectoryAction 的 Trajectory 元素中。 Trajectory 元素使用折线以世界位置格式定义了相关车辆的运动。将 TrajectoryFollowingMode 设置为 position 会强制参与者严格遵循指定的轨迹。导出的轨迹不包含时间维度。

<PrivateAction>
  <RoutingAction>
    <FollowTrajectoryAction>
       <Trajectory closed="false" name="Trajectory1">
         <Shape>
           <Polyline>
             <Vertex time="1">
               <Position>
                 <WorldPosition h="0" p="0" r="0" x="5" y="0" z="0"/>
               </Position>
             </Vertex>
             <Vertex time="2">
               <Position>
                 <WorldPosition h="0" p="0" r="0" x="10" y="0" z="0"/>
               </Position>
             </Vertex>
           </Polyline>
         </Shape>
      </Trajectory>
      <TimeReference/>      
      <TrajectoryFollowingMode followingMode="position"/>
    </FollowTrajectoryAction>
  </RoutingAction>
</PrivateAction>

注意

  • 该应用程序在驾驶场景中指定的航点之间插值生成平滑的轨迹，以便在输出的ASAM OpenSCENARIO文件中为导出的参与者生成平滑的轨迹。

  • 该应用程序不支持将参与者的反向航点导出到ASAM OpenSCENARIO文件中。

速度

当场景中的车辆速度发生变化时，该应用程序将此信息导出到输出的ASAM OpenSCENARIO文件中的 SpeedAction 元素中。该应用程序通过将 SpeedActionDynamics 元素的 dynamicsDimension 和 dynamicsShape 属性分别设置为 rate 和 linear，将速度变化导出为朝目标速度的恒定变化率。

<PrivateAction>
  <LongitudinalAction>
    <SpeedAction>
      <SpeedActionDynamics dynamicsDimension="rate" dynamicsShape="linear" value="5"/>
      <SpeedActionTarget>
        <AbsoluteTargetSpeed value="26"/>
      </SpeedActionTarget>
    </SpeedAction>
  </LongitudinalAction>
</PrivateAction>

等待时间

当驾驶场景中的参与者使用 wait (s) 参数时，该应用程序使用触发执行下一个 SpeedAction 的 condition 元素的 delay 属性导出此信息。

在此示例代码中，自我车辆在指定的航点停留 0.5 秒，然后继续向前行驶。事件 DS_Event_Ego4 指定自我车辆停在一个静止位置。代码使用 DS_Cond5 条件的 delay 属性表示等待时间信息，该条件在 DS_Event_Ego4 之后触发事件。因此，自我车辆在停下后等待 0.5 秒才开始再次移动。

<Event name="DS_Event_Ego4" priority="overwrite">
  <Action name="DS_Action_Ego4">
    <PrivateAction>
      <LongitudinalAction>
        <SpeedAction>
          <SpeedActionDynamics dynamicsDimension="rate" dynamicsShape="linear" value="-0.54932"/>
          <SpeedActionTarget>
            <AbsoluteTargetSpeed value="0"/>
          </SpeedActionTarget>
        </SpeedAction>
      </LongitudinalAction>
    </PrivateAction>
  </Action>
  <StartTrigger>
    <ConditionGroup>
      <Condition conditionEdge="none" delay="0" name="DS_Cond4">
        <ByEntityCondition>
          <TriggeringEntities triggeringEntitiesRule="any">
            <EntityRef entityRef="Ego"/>
          </TriggeringEntities>
          <EntityCondition>
            <ReachPositionCondition tolerance="2.0">
              <Position>
                <WorldPosition x="57" y="0" z="0"/>
              </Position>
            </ReachPositionCondition>
          </EntityCondition>
        </ByEntityCondition>
      </Condition>
    </ConditionGroup>
  </StartTrigger>
</Event>
<Event name="DS_Event_Ego5" priority="overwrite">
  <Action name="DS_Action_Ego5">
    <PrivateAction>
      <LongitudinalAction>
        <SpeedAction>
          <SpeedActionDynamics dynamicsDimension="rate" dynamicsShape="linear" value="0.77556"/>
          <SpeedActionTarget>
            <AbsoluteTargetSpeed value="2.5926"/>
          </SpeedActionTarget>
        </SpeedAction>
      </LongitudinalAction>
    </PrivateAction>
  </Action>
  <StartTrigger>
    <ConditionGroup>
      <Condition conditionEdge="none" delay="0.5" name="DS_Cond5">
        <ByValueCondition>
          <StoryboardElementStateCondition state="completeState" storyboardElementRef="DS_Action_Ego4" storyboardElementType="action"/>
        </ByValueCondition>
      </Condition>
    </ConditionGroup>
  </StartTrigger>
</Event>

参与者生成和消失

当您启用 Actor spawn and despawn 参数，并为 Entry Time (s) 和 Exit Time (s) 参数设置有效值时，该应用程序将使用VisibilityAction 元素导出此信息。

<PrivateAction>
  <VisibilityAction graphics="true" sensors="true" traffic="true"/>
</PrivateAction>
