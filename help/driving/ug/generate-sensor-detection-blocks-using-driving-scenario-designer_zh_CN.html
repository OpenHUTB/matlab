使用Driving Scenario Designer生成传感器模块

This example uses:

  • Automated Driving Toolbox
  • Simulink
Open Script

这个示例展示了如何使用Driving Scenario Designer 应用程序来更新Simulink®模型中的雷达和相机传感器。Driving Scenario Designer应用程序可以快速而交互地生成多个传感器配置。然后，您可以将这些生成的传感器配置应用于现有的Simulink模型中，以更全面地测试您的驾驶算法。

检查和模拟模型

此示例中使用的模型实现了自动紧急制动 （AEB） 传感器融合算法，并配置为模拟行人碰撞场景。有关此模型的更多详细信息，请参阅 Autonomous Emergency Braking with Sensor Fusion 示例。 打开模型。

open_system('AEBTestBenchExample')

用于测试算法的驾驶场景和传感器检测生成器位于 Vehicle Environment > Actors and Sensor Simulation 子系统中.。打开此子系统。

open_system('AEBTestBenchExample/Vehicle and Environment/Actors and Sensor Simulation')

Scenario Reader 模块从指定的Driving Scenario Designer文件中读取参与者和道路信息。该模块输出非自车的参与者。然后，这些参与者被传递给 Driving Radar Data Generator 和 Vision Detection Generator 传感器模块。在仿真过程中，这些模块会生成非自车参与者的检测结果。

在Bird's-Eye Scope上进行仿真和可视化。 在模型工具栏上，在“Review Results”下点击“Bird's-Eye Scope”。在作用域中，点击“Find Signals”，然后点击“Run”来运行仿真。在这个场景中，AEB模型使得自车及时制动以避免与正在过马路的行人儿童发生碰撞。

在这个示例中，您使用Driving Scenario Designer应用程序中创建的新传感器替换了模型中的现有传感器。

Load Scenario in App

该模型使用基于预构建的 Euro NCAP 测试协议方案之一的驾驶方案。您可以从驾驶方案设计器应用加载这些方案。有关这些方案的更多详细信息，请参阅Euro NCAP Driving Scenarios in Driving Scenario Designer。将方案文件加载到应用程序中。drivingScenarioDesigner('AEB_PedestrianChild_Nearside_50width_overrun.mat')

若要在应用中模拟方案，请单击“运行”。在应用程序模拟中，与模型模拟不同，自我车辆与行人相撞。该应用程序使用预定义的自我车辆轨迹，而该模型使用 AEB 算法来控制轨迹并导致自我车辆制动。

Load Sensors

加载的场景文件仅包含场景中的道路和参与者信息。传感器信息保存在一个单独的文件中。要将这些传感器加载到场景中，在应用程序工具栏上选择 Open > Sensors. 打开位于示例文件夹中的 AEBSensor.mat 文件。或者，从您的MATLAB根文件夹中导航到并打开此文件：

 matlabroot/examples/driving/data/AEBSensors.mat.

雷达传感器安装在ego车辆的前保险杠上。摄像头传感器安装在自我车辆的前窗上。

Update Sensors

通过更改雷达和摄像头传感器在自我车辆上的位置来更新它们。

  1. 在 Sensor Canvas上, 点击并拖动雷达传感器到预定义的Front Window位置。

  2. 点击并拖动相机传感器到预定义的Front Bumper（前保险杠）位置。在这个预定义位置上，应用程序将相机从短程传感器更新为长程传感器。

  3. （可选）在左窗格中的传感器选项卡上，尝试修改相机和雷达Sensors的参数。例如，您可以更改检测概率或精度和噪声设置。

  4. 将此新方案和传感器配置的副本保存到可写位置。

有关在应用中使用传感器的更多详细信息，请参阅 Create Driving Scenario Interactively and Generate Synthetic Sensor Data。此图显示了更新的传感器配置示例。

Export Scenario and Sensors to Simulink

要为场景及其传感器生成 Simulink 块，请在应用程序 toolstrip 上，选择Export > Export Simulink Model. 此模型显示从应用导出的示例块。
open_system('AEBGeneratedScenarioAndSensors')

如果未对场景中的道路和参与者进行任何更改，则场景读取器块将读取与 AEB 模型中使用的相同道路和执行组件数据。驾驶雷达数据生成器和视觉检测生成器模块对您在其中创建的雷达和摄像头进行建模。

将导出的场景和传感器复制到现有模型中

将 AEB 模型中的方案和传感器替换为新生成的方案和传感器。即使您没有修改道路和参与者并从同一方案文件中读取数据，替换现有方案读取器块仍然是最佳做法。使用此生成的块可使方案和传感器的总线名称在数据在它们之间传递时保持一致。

首先，在 AEB 模型中，重新打开Vehicle Environment > Actors and Sensor Simulation 子系统。open_system('AEBTestBenchExample/Vehicle and Environment/Actors and Sensor Simulation')

接下来，若要使用生成的方案和传感器块处理方案和传感器块，请执行以下步骤：

  1. 删除现有的场景读取器、驱动雷达数据生成器和视觉检测生成器块。不要删除输入到方案读取器块或从传感器块输出的信号线。或者，断开这些块的连接而不删除它们，并将它们注释出模型。使用此选项，您可以将现有块与新块进行比较，并在需要时恢复。选择每个块。然后，在“阻止”选项卡上，选择“注释掉”。

  2. 将块从生成的模型复制到 AEB 模型中。

  3. 打开复制的场景读取器块，并将Source of ego vehicle 参数设置成 Input port。点击 OK. AEB模型在“Pack Ego Actor”块中定义了自车，您需要将其连接到“Scenario Reader”块的“Ego Vehicle”端口。

  4. 将现有信号线连接到复制的模块。若要清理模型的布局，请在模型的Format选项卡上，选择Auto Arrange。

  5. 验证更新的子系统框图是否类似于预先存在的框图。然后，保存模型，或将模型的副本保存到可写位置。

模拟更新后的模型

若要可视化更新的方案模拟，请重新打开Bird's-Eye Scope，单击“ Find Signals”，然后单击 Run 。有了这个更新的传感器配置，自我车辆不会及时刹车。

若要尝试不同的传感器配置，请在应用中重新加载方案和传感器，导出新方案和传感器，然后将其复制到 AEB 模型中。
