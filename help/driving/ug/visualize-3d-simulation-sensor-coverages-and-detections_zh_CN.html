可视化虚幻引擎模拟环境中的传感器数据

此示例使用：

  • Automated Driving Toolbox
  • Simulink
打开脚本

此示例演示如何在 3D 仿真环境中可视化从高保真雷达和激光雷达传感器获得的传感器覆盖范围和检测结果。在此示例中，您将学习如何：

  1. 配置 Simulink® 模型以在 3D 环境中进行仿真。此环境是使用Epic Games®的虚幻引擎®渲染的。

  2. 从使用Driving Scenario Designer 应用程序创作的场景中读取地面实况数据和车辆轨迹，然后在 Simulink 模型中重新创建此场景。

  3. 使用Simulation 3D Probabilistic Radar和Simulation 3D Lidar激光雷达模块将雷达和激光雷达传感器添加到这些模型中。

  4. 在Bird's-Eye Scope中可视化驾驶场景和生成的传感器数据

您可以使用这些可视化和传感器数据来测试和改进您的自动驾驶算法。您还可以扩展此示例以融合检测并可视化对象跟踪结果，如在 Simulink 中使用合成雷达和视觉数据的传感器融合示例中所示。

检查长方体驾驶方案

在此示例中，基本实况（道路、车道和参与者）和车辆轨迹来自Driving Scenario Designer 应用中创作的方案。在此应用程序中，车辆和其他参与者表示为称为长方体的简单框形。有关创作长方体方案的更多详细信息，请参阅以交互方式创建驾驶方案和生成合成传感器数据示例。

在应用程序中打开长方体驾驶方案文件。

drivingScenarioDesigner('StraightRoadScenario.mat')

在应用中，运行方案模拟。在这种情况下，自我车辆（蓝色汽车）以恒定速度沿着笔直的道路向北行驶。在相邻的车道上，一辆橙色汽车以略高的恒定速度向北行驶。在对面的车道上，一辆黄色卡车以匀速向南行驶。

创作稍后在 3D 模拟环境中重新创建的驾驶场景时，必须使用与默认 3D 场景中的道路网络相同的道路网络。否则，在重新创建的场景中，车辆和传感器的位置不准确。此驾驶场景使用 Straight Road 场景的再现。要选择 3D 场景的不同长方体版本，请在应用程序工具行程中，选择 Open > Prebuilt Scenario > Simulation3D，然后从可用场景中进行选择。并非所有 3D 场景在应用程序中都有相应的版本。

  • 有关支持的场景列表以及有关每个场景的其他详细信息，请参阅驾驶方案设计器中的 3D 模拟场景的长方体版本。

  • 要为应用程序中不可用的场景生成车辆轨迹，请改用为虚幻引擎模拟选择航点示例中描述的过程。

长方体方案中车辆的尺寸也必须与预定义的 3D 模拟车辆类型之一的尺寸匹配。在应用程序工具之旅的 3D Display下，Use 3D Simulation Actor Dimensions 选项将每辆长方体车辆设置为具有 3D 车辆类型的尺寸。在此方案中，车辆具有这些 3D 显示类型和相应的车辆尺寸。

  • Ego Vehicle — Sedan vehicle dimensions

  • Vehicle in Adjacent Lane — Muscle Car vehicle dimensions

  • Vehicle in Opposite Lane — Box Truck vehicle dimensions

要将车辆更改为其他显示类型，请在应用程序左侧窗格的Actor选项卡上，更新该车辆的3D显示类型参数。要更改车辆的颜色，请选择所选车辆旁边的色块，然后选择新颜色。

要预览车辆在 3D 环境中的显示方式，请使用应用程序中提供的 3D 显示窗口。在应用程序工具行程中，选择3D Display > View Simulation in 3D Display模拟，然后重新运行模拟。

打开3D仿真模型

这个例子中使用的模型重新创建了长方体驾驶场景。该模型还定义了高保真度的传感器，可以从环境中生成合成的检测结果。打开该模型。

open_system('Visualize3DSimulationSensorCoveragesDetections')

检查场景配置

Simulation 3D Scene Configuration 块用于配置模型在3D环境中进行仿真。

  • 场景名称参数设置为默认的Straight road。该场景对应于应用程序场景文件中定义的长方体版本。

  • 场景视图参数设置为Ego Vehicle（自我车辆）。在仿真过程中，3D仿真窗口将从自我车辆后方显示场景。

Scenario Reader模块从应用程序场景文件中读取地面真实数据（道路边界、车道标线和参与者姿态）。 Bird's-Eye Scope 可视化这些地面真实数据，而不是3D仿真环境的地面真实数据。要在长方体和3D仿真环境中使用相同的场景，两个环境的地面真实数据必须匹配。如果您正在创建一个新的场景，您可以生成一个Scenario Reader模块，用于从您的场景文件中读取数据。首先，在 Driving Scenario Designer 应用程序中打开场景文件。然后，在应用程序工具栏上，选择 Export > Export Simulink Mode。如果您更新了场景，您无需生成新的Scenario Reader模块。

Simulation 3D Scene Configuration模块和Scenario Reader模块的采样时间参数都设置为0.1。此外，所有其他3D仿真车辆和传感器模块都从Simulation 3D Scene Configuration模块继承其采样时间。通过在整个模型中设置单一的采样时间，Bird's-Eye Scope以恒定的速率显示来自所有模块的数据。如果地面真实数据和传感器数据具有不同的采样时间，则范围会在不同的时间间隔内可视化它们。这个过程会导致地面真实数据和传感器数据的可视化闪烁。

检查车辆配置

Simulation 3D Vehicle with Ground Following 模块指定了3D仿真环境中车辆的外观和轨迹。每辆车都是与 Driving Scenario Designer 应用程序场景文件中定义的车辆直接对应的。

在3D环境中，车辆的位置以世界坐标表示。然而，Scenario Reader模块以自车坐标系输出非自车角色的姿态。 Vehicle To World 模块将这些非自车角色的姿态转换为世界坐标。由于自车以世界坐标输出，因此对于自车来说，这种转换是不必要的。有关车辆和世界坐标系的更多详细信息，请参阅 Coordinate Systems in Automated Driving Toolbox。

车辆原点的位置在长方体和3D场景之间有所不同。

  • 在长方体场景中，车辆原点位于地面上，位于后轴的中心位置。

  • 在3D场景中，车辆原点位于地面上，位于车辆的几何中心位置。

Cuboid To 3D Simulation 模块将长方体原点位置转换为3D仿真原点位置。在这些模块的转换参数中使用的 ActorID 指定了要转换的每个车辆的特定 ActorID ，Scenario Reader模块在其 Actors 输出端口输出 ActorID 值。在 Driving Scenario Designer 应用程序中，您可以在Actors选项卡中的角色选择列表中找到相应的ActorID值。每个车辆的ActorID是冒号前面的值。

每个Cuboid To 3D Simulation模块输出X、Y和偏航角（Yaw）值，直接输入到相应的车辆模块中。在3D仿真环境中，3D场景的地面地形决定了车辆的Z轴位置（高度）、滚动角度（Roll）和俯仰角度（Pitch）。

在每个Simulation 3D Vehicle with Ground Following模块中，Type 参数对应于应用程序中为该车辆选择的3D显示类型。此外，Color 参数对应于应用程序中指定的车辆颜色。为了在 Bird's-Eye Scope 和 3D 仿真窗口之间保持类似的车辆可视化效果，指定的类型和颜色必须匹配。要在应用程序中更改车辆的颜色，在 Actors 选项卡上，点击角色选择列表中角色名称右侧的颜色块。选择与Simulation 3D Vehicle with Ground Following模块的 Color 参数中可用颜色最接近的颜色。

检查传感器配置

该模型包括两个传感器块，具有默认参数设置。这些块从3D模拟环境中生成检测结果。

  • Simulation 3D Probabilistic Radar 传感器块基于统计模型生成物体检测结果。该传感器安装在自动驾驶车辆的前保险杠上。

  • Simulation 3D Lidar 传感器块以点云形式生成检测结果。该传感器安装在自动驾驶车辆的车顶中央。

虽然您可以在 Driving Scenario Designer 应用程序中指定传感器并将其导出到Simulink，但导出的模块与3D模拟环境不兼容。您必须直接在模型中指定3D模拟传感器。

模拟和可视化场景

在模拟过程中，您可以在3D模拟窗口和 Bird's-Eye Scope内可视化场景。

首先，打开作用域。在Simulink工具栏上，在 Review Results 下，点击 Bird's-Eye Scope。然后，点击 Find Signals以查找作用域可以显示的信号。

要运行模拟，请在模型或作用域中点击 Run 。当模拟开始时，3D模拟窗口可能需要几秒钟来初始化，尤其是在Simulink会话中首次运行时。当此窗口打开时，它会显示具有高保真度图形的场景，但不显示检测或传感器覆盖范围。

Bird's-Eye Scope 通过使用长方体表示来显示检测和传感器覆盖范围。雷达覆盖区域和检测结果显示为红色。激光雷达（lidar）覆盖区域显示为灰色，其点云检测结果以parula色图显示。

该模型以每个墙钟秒的0.5秒的速度运行模拟。要调整速度，请从Simulink工具栏中选择 Run > Simulation Pacing，然后移动滑块以增加或减少模拟的速度。

修改驾驶场景

当修改驾驶场景时，您可能需要根据您的更改在 Driving Scenario Designer 应用程序中更新场景、Simulink模型或两者同时更新，具体取决于您的修改内容。

  • 修改道路网络 - 在应用程序中，从 Simulation3D 文件夹中选择一个新的预构建场景。不要修改这些道路网络，否则道路将与所选的3D场景中的道路不匹配。在模型中，在Simulation 3D Scene Configuration模块中，选择对应的场景名称作为Scene name参数。

  • 修改车辆轨迹 - 在应用程序中修改车辆轨迹并重新保存场景。在模型中，您无需更新任何内容来适应此更改。Scenario Reader模块会自动获取这些更改。

  • 修改车辆外观 - 在应用程序中，更新车辆的颜色和3D显示类型参数。还要确保选择了3D Display > Use 3D Simulation Actor Dimensions选项。在模型中，更新相应的Simulation 3D Vehicle with Ground Following模块的颜色和类型参数。

  • 添加新车辆 - 在应用程序中，创建一个新的车辆，并指定轨迹、颜色和3D显示类型。在模型中，添加一个新的Simulation 3D Vehicle with Ground Following 模块和相应的 Cuboid To 3D Simulation 模块。将这些模块设置得与现有的非自车车辆类似。在Cuboid To 3D Simulation模块中，设置新车辆的ActorID 。

  • 设置新的自车 - 在应用程序中，转到 Actors 选项卡，选择要设置为自车的车辆，然后点击 Set As Ego Vehicle。在模型中，更新Cuboid To 3D Simulation模块中用于转换参数的 ActorID，以反映哪辆车是新的自车。在传感器模块中，设置 Parent name 参数，使传感器安装在新的自车上。

  • 修改或添加传感器 - 在应用程序中，您不需要进行任何更改。在模型中，修改或添加传感器模块。当添加传感器模块时，将所有传感器的 Parent name 设置为自车。

要在鸟瞰图中可视化任何更新的方案，必须再次单击查找信号。如果您修改方案或只想可视化传感器数据，请考虑在仿真期间关闭 3D 窗口。在模拟 3D 场景配置块中，清除显示 3D 模拟窗口参数。
