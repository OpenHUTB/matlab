自动驾驶工具箱中虚幻引擎模拟的坐标系

自动驾驶工具箱™使你能够在使用Epic Games®虚幻引擎®的虚拟环境中模拟驾驶算法。通常，此环境中使用的坐标系遵循自动驾驶工具箱中的坐标系中所述的约定。但是，在此环境中进行仿真时，了解坐标系的特定差异和实现细节非常重要。

世界坐标系

与其他自动驾驶工具箱功能一样，仿真环境使用 ISO 8855 中定义的右手笛卡尔世界坐标系。虚拟城市场景的以下 2D 顶视图图像显示了场景的 X 和 Y 坐标。

在此坐标系中，当向 X 轴的正方向看时，正 Y 轴指向左。正 Z 轴从地面向上指向。当分别朝 Z、Y 和 X 轴的正方向观察时，偏航角、俯仰角和横滚角是顺时针正的。如果从 2D 自上而下的视角查看场景，则偏航角为逆时针正，因为您是在 Z 轴的负方向上查看场景。

在场景中放置车辆

车辆放置在场景的世界坐标系中。该图显示了在具有地面跟随模块的模拟 3D 车辆中指定 X、Y 和偏航端口如何确定它们在场景中的位置。

地面的仰角和倾斜角决定了车辆的 Z 轴、横滚角和俯仰角。

与虚幻编辑器世界坐标的区别

虚幻®编辑器使用左手世界笛卡尔坐标系，其中正Y轴指向右侧。如果要从虚幻编辑器坐标系转换为3D环境的坐标系，则必须翻转Y轴和俯仰角的符号。两个坐标系中的 X 轴、Z 轴、横滚角和偏航角相同。

车辆坐标系

车辆坐标系基于世界坐标系。在此坐标系中：

  • X 轴从车辆指向前方。

  • Y 轴指向车辆左侧。

  • Z 轴从地面向上指向。

  • 横滚、俯仰和偏航分别在向前看 X、Y 和 Z 轴时顺时针正向。与世界坐标系一样，当从上到下查看车辆时，偏航角是逆时针正的。

车辆原点在地面上，在车辆的几何中心。在此图中，蓝点表示车辆原点。

在车辆上安装传感器

将传感器块（如模拟 3D 相机模块）添加到模型时，可以将传感器安装到预定义的车辆位置，例如根中心的前保险杠。这些安装位置位于车辆坐标系中。指定与这些位置的偏移时，将从安装位置的原点偏移，而不是从车辆原点偏移。

这些方程定义了具有位置（X、Y、Z）和方向（横滚、俯仰、偏航）的传感器的车辆坐标：

  • (X, Y, Z) = (Xmount + Xoffset, Ymount + Yoffset, Zmount + Zoffset)

  • (Roll, Pitch, Yaw) = (Rollmount + Rolloffset, Pitchmount + Pitchoffset, Yawmount + Yawoffset)

“安装”变量是指相对于车辆原点的预定义安装位置。您可以在传感器模块的安装位置参数中定义这些安装位置。

“偏移”变量是指这些安装位置的偏移量。您可以在传感器模块的相对平移 [X， Y， Z] （m） 和相对旋转 [横滚、俯仰、偏航] （度） 参数中定义这些偏移。

例如，考虑安装在后保险杠位置的传感器。相对于车辆原点，传感器的方向为 （0， 0， 180）。换句话说，当从上往下看车辆时，传感器的偏航角逆时针旋转180度。

要将传感器进一步指向右侧 90 度，您需要将相对旋转 [横滚、俯仰、偏航]（度）参数设置为 [0，0，90]。换句话说，传感器相对于车辆原点逆时针旋转 270 度，但相对于预定义后保险杠位置的原点仅逆时针旋转 90 度。

与长方体车辆起源的区别

在长方体模拟环境中，如长方体场景模拟中所述，原点位于地面上，位于车辆后轴中心下方。

长方体车辆起源 3D 模拟车辆起源

如果要在坐标系之间转换传感器位置，则需要使用长方体转 3D 模拟块来考虑原点上的这种差异。有关使用此块的示例模型，请参阅高速公路车道跟随。

与虚幻编辑器车辆坐标的区别

虚幻编辑器使用左手笛卡尔车辆坐标系，其中正Y轴指向右侧。如果要从虚幻编辑器坐标系转换为虚幻引擎环境的坐标系，则必须翻转Y轴和俯仰角的符号。两个坐标系中的 X 轴、Z 轴、横滚角和偏航角相同。
