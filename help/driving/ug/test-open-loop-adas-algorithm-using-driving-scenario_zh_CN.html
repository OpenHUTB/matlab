使用驾驶场景测试开环ADAS算法

This example uses:

  • Automated Driving Toolbox
  • Simulink
Open Live Script

这个示例演示了如何在Simulink中测试一个开环ADAS（高级驾驶员辅助系统）算法。在开环ADAS算法中，自车的行为是预定义的，并且在仿真过程中随着场景的发展不会改变。

测试该算法，您需要使用从Driving Scenario Designer 应用程序保存的驾驶场景。在这个示例中，您可以使用Scenario Reader块读取场景，并在Bird's-Eye Scope上可视化场景和传感器检测结果。

检查驾驶场景

这个示例使用的驾驶场景是基于Driving Scenario Designer应用程序中的预建场景之一。有关这些场景的更多详细信息，请参阅Prebuilt Driving Scenarios in Driving Scenario Designer。

在应用程序中打开场景文件。

drivingScenarioDesigner('LeftTurnScenario.mat')

要模拟这个场景，点击Run。在这个场景中，自动驾驶车辆向北行驶，并直接通过一个十字路口。一辆来自十字路口左侧的车辆左转，并最终停在自动驾驶车辆前方。

自我车辆具有以下传感器：

  • 用于生成物体检测的前置雷达

  • 前置摄像头和后置摄像头，用于生成物体和车道边界检测

  • 屋顶中心的激光雷达，用于生成场景的点云数据

  • 两个超声波传感器，分别面向前方和前方，用于生成物体检测

检查模型

此示例中的模型是通过选择Export > Export Simulink Model从应用程序生成的。在模型中，Scenario Reader从场景文件中读取参与者和道路，并输出非自我参与者和车道边界。打开模型。

open_system('OpenLoopWithScenarios.slx')

在 Scenario Reader 模块中，Driving Scenario Designer file name参数指定了场景文件的名称。您可以指定一个位于MATLAB®搜索路径上的场景文件，例如此示例中使用的场景文件，或者指定场景文件的完整路径。或者，您可以通过将drivingScenario设置为From workspace，然后将MATLAB或模型工作区变量名称设置为有效的drivingScenario对象工作区变量的名称，来指定一个驾驶场景对象。

Scenario Reader模块输出场景中非自我参与者的姿势以及自我车辆的左车道和右车道边界。要输出自我车辆行驶的道路的所有车道边界，选择Lane boundaries to output 参数的相应选项。

参与者、车道边界和自我车辆姿势被传递到包含传感器块的子系统。打开子系统。

open_system('OpenLoopWithScenarios/Detection Generators')

Driving Radar Data Generator, Vision Detection Generator, Lidar Point Cloud Generator, 和 Ultrasonic Detection Generator 模块会从场景中生成合成的检测数据。您可以将这些传感器数据融合起来生成轨迹，例如在Sensor Fusion Using Synthetic Radar and Vision Data in Simulink中所示。

该模型中传感器模块的输出以车辆坐标为单位，其中：

  • X轴从自我车辆指向前方。

  • Y轴指向自我车辆的左侧。

  • 原点位于自我车辆后轴的中心。

由于该模型是开环模型，随着仿真的进行，自车的行为不会改变。因此，Source of ego vehicle 参数被设置为Scenario，该模块从场景文件中读取预定义的自车姿态和轨迹。对于车辆控制器和其他闭环模型，将Source of ego vehicle参数设置为“Input port”。通过此选项，您可以将在模型中定义的自车作为输入传递给场景读取器模块。有关示例，请参阅Test Closed-Loop ADAS Algorithm Using Driving Scenario.。

可视化仿真

要可视化场景和传感器检测结果，请使用 Bird's-Eye Scope。在 Simulink 工具栏上，点击 Review Results，然后选择Bird's-Eye Scope。然后，在 Scope 中，点击 Find Signals 并运行仿真。

更新仿真设置

该模型使用默认的仿真停止时间为10秒。由于场景的长度约为5秒，即使场景结束后，仿真仍会在 Bird's-Eye Scope 中继续运行。为了使仿真停止时间与场景停止时间同步，在 Simulink 模型工具栏上，将仿真停止时间设置为5.2秒，这是应用场景的确切停止时间。运行仿真后，应用程序会在场景画布的右下角显示该值。

如果 Bird's-Eye Scope 中的仿真运行过快，您可以使用仿真节奏来减慢仿真速度。在 Simulink 工具栏上，选择 Run > Simulation Pacing。选中“Enable pacing to slow down simulation”复选框，并将仿真时间减少到略低于每秒钟的墙钟时间1秒，例如0.8秒。然后，在 Bird's-Eye Scope 中重新运行仿真。
