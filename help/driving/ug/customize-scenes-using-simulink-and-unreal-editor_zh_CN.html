使用 Simulink 和虚幻编辑器自定义场景

Step 3 of 4 in Customize Unreal Engine Scenes for Automated Driving

如安装自定义场景支持包中所述，在安装虚幻引擎®4项目支持包的自动驾驶工具箱™界面后，您可以同时从虚幻®编辑器和Simulink®在自定义场景中进行模拟。通过使用此联合仿真框架，您可以将车辆和传感器添加到Simulink模型中，然后在自定义场景中运行此仿真。

要使用使用先前版本的支持包开发的项目，首先要将项目迁移到与当前支持的虚幻引擎版本兼容。请参阅使用先前支持包开发的迁移项目。

从Simulink打开虚幻编辑器

如果从MATLAB®或Simulink外部打开虚幻编辑器，则Simulink无法与编辑器建立连接。要建立此连接，必须从Simulink模型打开项目。

  1. 打开一个配置为在虚幻引擎环境中进行模拟的Simulink模型。至少，模型必须包含仿真3D场景配置块。例如，打开一个简单的模型，模拟车辆在笔直的高速公路上行驶。该模型在“使用虚幻引擎仿真环境设计车道标志检测器”实例中得到应用。

    openExample('driving/VisualPerceptionIn3DSimulationExample')
open_system('straightRoadSim3D')

  2. 在该模型的仿真3D场景配置块中，将场景源参数设置为虚幻编辑器。

  3. 在Project参数中，浏览包含要自定义的场景的项目文件。

    例如，这个示例路径指定了AutoVrtlEnv项目，该项目与虚幻引擎4项目支持包的自动驾驶工具箱接口一起安装。

    C:\Local\AutoVrtlEnv\AutoVrtlEnv.uproject
    这个示例路径指定了一个自定义项目。
    Z:\UnrealProjects\myProject\myProject.uproject

  4. 点击Open Unreal Editor. 虚幻编辑器打开并加载项目中的场景。

第一次从Simulink打开虚幻编辑器时，您可能会被要求重建UE4Editor DLL文件或AutoVrtlEnv模块。单击Yes以重新生成这些文件或模块。编辑器还会提示有新的插件可用。单击Manage Plugins并验证是否安装了MathWorks Interface插件。这个插件是MathWorksSimulation.uplugin，是从MATLAB支持包中复制过来的插件文件。

关于名称为“_BuiltData”的文件的消息表示缺少相关级别的照明数据。在发布可执行文件之前，重建关卡照明。

如果您收到需要重新构建照明的警告，请从编辑器窗口上方的工具栏中选择Build > Build lighting Only。编辑器在您第一次打开场景或向场景添加新元素时发出此警告。要使用自动驾驶工具箱中安装的AutoVrtlEnv照明，请参阅在自定义场景中使用AutoVrtlEnv项目照明。

父角色蓝图

注意

如果您正在使用AutoVtrlEnv或RRScene项目中的场景，它是虚幻引擎4项目支持包的自动驾驶工具箱接口的一部分，请跳过本节。但是，如果您基于此项目中的场景之一创建新场景，则必须完成此部分。

当你第一次从Simulink中打开一个自定义场景时，你需要将这个项目与自动驾驶工具箱中使用的Sim3dLevelScriptActor关卡蓝图关联或重新定义。关卡蓝图控制对象一旦放置在虚幻引擎环境中如何与之交互。如果项目没有重新定义，Simulink将在模拟开始时返回一个错误。您必须分别表示自定义项目中的每个场景。

要绘制关卡蓝图，请遵循以下步骤:

  1. 在虚幻编辑器工具栏中，选择 Blueprints > Open Level Blueprint。

  2. 关卡蓝图窗口中，选择 File > Reparent Blueprint。

  3. 点击Sim3dLevelScriptActor蓝图。如果你没有看到Sim3dLevelScriptActor蓝图列表，使用以下步骤检查你是否已经安装并启用了MathWorksSimulation插件:

    1. 在虚幻编辑器工具栏中，选择 Settings > Plugins。

    2. 在Plugins窗口中，确认已安装的窗口中列出了MathWorks Interface插件。如果插件尚未启用，请选中enabled复选框。

      如果在此窗口中没有看到MathWorks接口插件，请重复安装自定义场景支持包下的步骤，并从Simulink重新打开编辑器。

    3. 关闭编辑器并从Simulink重新打开它。

  4. 关闭关卡蓝图窗口。

在虚幻编辑器创建或修改场景

从Simulink中打开编辑器后，您可以修改项目中的场景或创建新场景。

打开场景

在虚幻编辑器中，项目中的场景被称为关卡。关卡有多种类型，场景也有关卡类型的地图。

  • 从AutoVrtlEnv.uproject或RRScene.uproject文件打开预构建场景，在编辑器窗口下方的Content Browser窗格中，导航到Content > Maps文件夹。然后，选择与你想要修改的场景相对应的地图。

    这个表格显示了在虚幻编辑器中出现的AutoVrtlEnv项目中的地图名称。它还显示了它们对应的场景名称，因为它们出现在模拟3D场景配置块的场景名称参数中。

    虚幻编辑器地图 自动驾驶工具箱场景
    HwCurve Curved Road
    DblLnChng Double Lane Change
    BlackLake Open Surface
    LargeParkingLot Large Parking Lot
    SimpleLot Parking Lot
    HwStrght Straight Road
    USCityBlock US City Block
    USHighway US Highway

    注意

     AutoVrtlEnv.uproject 不包含 Virtual Mcity 场景。

    RRScene项目只包含一个场景:RRHighway。此场景用于公路车道跟随RoadRunner场景示例，并且不能从模拟3D场景配置块的场景名称参数中选择。

  • 要在您自己的项目中打开场景，请在Content Browser窗格中导航到包含场景的文件夹。

创建新场景

要在项目中创建一个新场景，从编辑器的左上角菜单中选择File > new Level。

或者，您可以从现有场景创建一个新场景。这项技术是有用的，例如，如果你想使用AutoVtrlEnv项目中的一个预构建场景作为创建自己的场景的起点。要将当前打开的场景的版本保存到项目中，从编辑器的左上角菜单中选择File > save Current As。新场景被保存到与现有场景相同的位置。

指定车辆轨迹

在你的场景中，你可以指定场景中的车辆可以遵循的轨迹路径点。

  • 如果你的场景是基于AutoVrtlEnv项目中的一个预构建场景，那么使用虚幻引擎仿真示例中选择路径点所描述的过程来指定路径点。这个例子展示了如何在预构建场景的2d自上而下的地图上交互式地绘制路径点。

  • 如果您的场景不是基于预构建的场景，那么在使用虚幻引擎模拟示例的选择路径点之前，您必须首先生成您的场景的地图。参见创建虚幻引擎场景的自上而下的地图。

向场景添加资源

在虚幻编辑器中，场景中的元素被称为资产。要将资源添加到场景中，您可以在底部的Content Browser窗格中浏览或搜索它们，并将它们拖到编辑器窗口中。

在AutoVrtlEnv项目中向场景添加资源时，您可以从与驾驶相关的资源库中进行选择。这些资源被构建为静态网格，并以前缀SM_开始。在Content Browser窗格中搜索这些对象。

例如，在AutoVrtlEnv项目中为场景添加一个停止标志。

  1. 在编辑器底部的Content Browser窗格中，导航到Content文件夹。

  2. 在搜索栏中搜索“SM_StopSign”。将停止标志从Content Browser拖到编辑窗口中。然后，您可以在编辑窗口或右边的Details窗格的Transform部分中更改停止标志的位置。

如果你想覆盖默认天气或在场景中使用增强的雾条件，添加Exponential Height Fog。

虚幻编辑器使用左z向上坐标系统，其中y轴指向右。自动驾驶工具箱使用右手z向上坐标系，其中y轴指向左边。当在场景中定位物体时，请记住这个坐标系的差异。在这两个坐标系中，y轴的正负号和俯仰角值是颠倒的。

有关修改场景和添加资产的更多信息，请参阅虚幻引擎4文档。

要将资产从AutoVrtlEnv项目迁移到您自己的项目文件中，请参阅虚幻引擎文档。

要从场景中获得语义分割数据，必须将模板id应用于添加到场景中的对象。有关详细信息，请参阅将标签应用于虚幻场景元素以进行语义分割和对象检测。

在自定义场景中使用AutoVrtlEnv项目照明

要使用自动驾驶工具箱中的AutoVrtlEnv项目附带安装的照明，请遵循以下步骤。

  1. 在World Settings选项卡, 清除 Force No Precomputed Lighting.

  2.  在 Build下，选择Lighting Quality > Production，以生产质量重建地图。重建大型地图需要时间。

运行仿真

通过运行测试模拟，验证Simulink模型和虚幻编辑器已配置为共同模拟。

  1. 在Simulink模型中，单击Run。

    因为场景的源是在虚幻编辑器中打开的项目，所以模拟不会开始。相反，您必须从编辑器启动模拟。

  2. 验证Simulink中的“诊断查看器”窗口是否显示此消息:

    In the Simulation 3D Scene Configuration block, you set the scene source to 'Unreal Editor'. In Unreal Editor, select 'Play' to view the scene.

    此消息确认Simulink已经在虚幻引擎环境中实例化了车辆和其他对象。

  3. 在虚幻编辑器中，点击播放。模拟在虚幻编辑器中当前打开的场景中运行。

    • 如果您的Simulink模型包含车辆，这些车辆将通过在编辑器中打开的场景驾驶。

    • 如果您的Simulink模型包括传感器，这些传感器从编辑器中打开的场景中捕获数据。

为了在模拟过程中控制场景的视图，在模拟3D场景配置模块中，从场景视图参数中选择车辆名称。要在模拟运行时更改场景视图，请使用编辑器中的数字键盘。该表显示了显示场景的相机的位置，相对于在场景视图参数中选择的车辆。

要重新启动模拟，请单击Simulink模型中的Run，等待直到Diagnostic Viewer显示确认消息，然后单击编辑器中的Play。如果在模型中开始模拟之前单击Play，则Simulink和虚幻编辑器之间的连接未建立，编辑器将显示一个空场景。

如果您正在共同模拟自定义项目，要启用数字键盘，请将DefaultInput.ini文件从支持包安装文件夹复制到自定义项目文件夹中。例如，从以下路径复制DefaultInput.ini:

C:\ProgramData\MATLAB\SupportPackages\<MATLABRelease>\toolbox\shared\sim3dprojects\driving\AutoVrtlEnv\Config
到:
C:\<yourproject>.project\Config

在根据模拟结果调整自定义场景后，您可以将场景打包到可执行文件中。有关详细信息，请参阅将自定义场景打包为可执行文件。
