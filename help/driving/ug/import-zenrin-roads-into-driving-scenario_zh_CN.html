将Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）导入到驾驶场景中。

由ZENRIN DataCom CO.，LTD.开发的Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) 1 是一个网络服务，可让您访问日本地区的地图数据。使用 Driving Scenario Designer 应用程序，您可以从 Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) 服务中导入地图数据，并将其用于生成驾驶场景中的道路。

此示例侧重于在应用程序中导入地图数据。或者，要将Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) 数据导入到 drivingScenario 对象中，请使用 roadNetwork 函数。

从 Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) 服务导入地图数据需要使用 Automated Driving Toolbox Importer for Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) Service。

设置Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）凭据

要访问Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）服务，您必须输入来自ZENRIN DataCom CO.，LTD的有效凭据。通过使用 zenrinCredentials 函数设置这些凭据。在MATLAB®命令提示符下，输入：

zenrinCredentials setup

在Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）凭据对话框中，输入有效的 Client ID 和 Secret Key。要在您的计算机上保存凭据以供将来的MATLAB会话使用，在对话框中选择 Save my credentials between MATLAB sessions，然后点击 OK。凭据现在已保存在您的计算机上的其余MATLAB会话中。

如果您需要更改凭据，可以通过使用 zenrinCredentials 函数删除它们并设置新的凭据。

指定地理坐标

要选择要导入的道路，您需要指定一个感兴趣的区域，以获取道路数据。为了定义这个感兴趣的区域，需要指定附近道路数据的纬度和经度坐标。您可以为单个点或一组点（例如构成驾驶路线的点）指定坐标。

指定驾驶路线的坐标。

  1. 加载由一系列纬度和经度坐标组成的驾驶路线。在MATLAB命令提示符处，输入以下命令：

    data = load('tokyoSequence.mat');
lat = data.latitude;
lon = data.longitude;

  2. 打开应用程序。

    drivingScenarioDesigner

  3. 在应用程序工具栏上，选择 Import ，然后选择 Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0)。如果您之前输入或保存了Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）凭据，则对话框直接打开到您可以指定地理坐标的页面。

  4. 选择 From Workspace ，然后选择用于路线坐标的变量。

    • 将 Latitude 参数设置为 lat。

    • 将 Longitude 参数设置为 lon。

    以下表格描述了指定纬度和经度坐标的完整选项列表。

    Specify Geographic Coordinates Parameter Value Description Latitude Parameter Value Longitude Parameter Value
    From Workspace

    请指定一组纬度和经度坐标，例如通过GPS获取的驾驶路线。这些坐标必须作为变量存储在MATLAB工作区中。

    工作区变量包含在范围[-90, 90]内的十进制值的向量。单位为度。

    纬度和经度必须具有相同的大小。选择纬度变量后，经度列表仅包括与您选择的纬度大小相同的变量。

    工作区变量包含在范围[-180, 180]内的十进制值的向量。单位为度。

    纬度和经度必须具有相同的大小。选择经度变量后，如果选择了不同大小的纬度变量，则对话框会清除您的经度选择。
    Input Coordinates

    指定一个地理点的纬度和经度坐标。

    		 十进制标量，范围为[-90, 90]。单位为度。 十进制标量，范围为[-180, 180]。单位为度。

选择包含道路的区域

在您指定纬度和经度坐标之后，对话框的 Select Region 部分会在地图上以橙色显示这些坐标。导入场景的地理参考点，即驾驶路线中的第一个坐标，也会显示出来。单击该点可显示或隐藏坐标数据。

这些坐标被连接成一条线。一个矩形感兴趣区域会围绕这些坐标显示。在对话框的下一页中，您可以根据至少部分位于该区域内的道路来选择要导入的道路。

您可以更改该区域的大小或移动它以选择不同的道路。要放大或缩小该区域，请使用地图显示右上角的按钮。

在坐标仍然包含在该区域内的情况下，单击 Next。

选择要导入的道路

在选择区域后，对话框的 Select Roads 部分会显示黑色的可选择道路。

使用所选区域，通过单击 Select Nearest Roads 来选择最靠近驾驶路线的道路。所选道路将叠加在驾驶路线上，并以蓝色显示。

以下是您可以从区域中选择道路的其他操作的描述。

Goal Action
从区域中选择单个道路。 单击各个道路以选择它们。
从区域中选择所有道路。 单击 Select All。
从区域中选择除了少数道路之外的所有道路。 单击 Select All，然后单击各个道路以取消选择。
选择最靠近指定坐标的区域中的道路。 单击 Select Nearest Roads。当您有一系列非稀疏坐标时，请使用此选项。如果您的坐标是稀疏的或者这些坐标的基础数据是稀疏的，那么应用程序可能无法选择最近的道路。
从区域中选择一部分道路，例如区域上半部分的所有道路。

在地图显示的左上角，单击“选择道路”按钮。然后，绘制一个围绕道路的矩形以进行选择。

  • 要从此选择中取消选择一部分道路，请单击“取消选择道路”按钮。然后，绘制一个围绕要取消选择的道路的矩形。

  • 要取消选择所有道路并重新开始，请单击 Deselect All。

注意

您选择的道路数量直接影响应用程序的性能。请选择您需要创建驾驶场景所需的最少道路。

导入道路

在仍然选择最靠近路线的道路的情况下，单击 Import。应用程序将导入Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）数据并生成道路网络。

为了与地理地图显示保持相同的对齐，Scenario Canvas 的X轴在底部，Y轴在左侧。在不从地图导入的驾驶场景中，X轴在左侧，Y轴在底部。这种对齐与Automated Driving Toolbox™的世界坐标系一致。

场景的原点对应于地理参考点，即驾驶路线中指定的第一个点。即使您从驾驶路线的末尾选择道路，原点仍然锚定在这个第一个点上。如果您使用 Input Coordinates 选项指定了单个地理点，则原点是该点。

默认情况下，禁用道路交互。禁用道路交互可以防止您意外修改网络，并通过隐藏道路中心来减少视觉杂乱。如果您想修改道路，请在 Scenario Canvas 的左下角单击“配置场景画布”按钮。然后，选择 Enable road interactions。

注意

在某些情况下，应用程序无法导入所有选择的道路。应用程序会暂停导入，并在对话框中以红色突出显示无法导入的道路。要继续导入所有其他选择的道路，请单击 Continue。

将导入的道路与地图数据进行比较

应用程序中生成的道路网络与实际的Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）道路网络有几个区别。例如，当Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）服务使用范围信息时，例如指定具有两到三条车道或3-5.5米宽的道路时，生成的道路网络使用标量值。

有关从Zenrin Japan Map API 3.0（Itsumo NAVI API 3.0）数据生成的道路网络的限制的更多详细信息，请参阅 Driving Scenario Designer 应用程序参考页面的 Limitations 部分。

保存场景

保存场景文件。保存场景后，您无法将其他 Zenrin Japan Map API 3.0 (Itsumo NAVI API 3.0) 道路导入到其中。相反，您需要创建一个新的场景并导入新的道路网络。

现在，您可以向场景中添加角色和传感器，为测试驾驶算法生成合成车道和物体检测，或将场景导入到Simulink®中。
