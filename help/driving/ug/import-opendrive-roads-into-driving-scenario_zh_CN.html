将ASAM OpenDRIVE道路导入驾驶场景

ASAM OpenDRIVE®是一种开放的文件格式，可以用于指定大型和复杂的道路网络。使用 Driving Scenario Designer 应用程序，您可以将ASAM OpenDRIVE文件中的道路和车道导入到驾驶场景中。然后，您可以向场景中添加参与者和传感器，并生成用于测试在MATLAB®中开发的驾驶算法的合成车道和物体检测。或者，要测试在Simulink®中开发的驾驶算法，您可以使用 Scenario Reader 块将道路网络和参与者读入模型中。

该应用程序支持从OpenDRIVE®文件版本1.4和1.5以及ASAM OpenDRIVE文件版本1.6导入道路网络。

要将ASAM OpenDRIVE道路和车道导入到 drivingScenario 对象而不是应用程序中，请使用 roadNetwork 函数。

导入ASAM OpenDRIVE文件

要将 ASAM OpenDRIVE 文件导入 Driving Scenario Designer 应用程序，请按照以下步骤操作：

  1. 打开 Driving Scenario Designer 应用程序。在MATLAB命令提示符处输入：

    drivingScenarioDesigner

  2. 在应用程序工具栏上，单击 Import > ASAM OpenDRIVE File。

  3. 在 ASAM OpenDRIVE 导入对话框中，浏览有效的ASAM OpenDRIVE文件(.xodr或 .xml类型)。

    在此示例中，您导航到此文件，其中 matlabroot 是您的MATLAB文件夹的根目录：

    matlabroot/examples/driving/data/roundabout.xodr

    注意

    您不能将 ASAM OpenDRIVE 道路网络导入到现有的驾驶场景中。如果您尝试这样做，应用程序会提示您保存现有的场景。

  4. （可选）如果您不想看到导入过程中可能出现的任何错误或警告，请清除 Show errors and warnings 参数。默认情况下，此参数已选中。

  5. 点击 Import。如果您选择了 Show errors and warnings 参数，则“导入ASAM OpenDRIVE结果”对话框会显示导入过程中出现的任何警告和错误。您可以将这些警告和错误复制到文本文件中。然后，关闭对话框。

应用程序的 Scenario Canvas 将显示导入的道路网络。

注意

从R2021b版本开始，ASAM OpenDRIVE导入功能提供了功能和视觉上的改进，以及一些额外的限制。

  • 在导入ASAM OpenDRIVE道路网络后，您现在可以向场景中添加新的道路，并导出MATLAB函数。

  • 您现在可以导入具有多个车道规格的道路。

  • 导入的道路显示了以前未显示的边界线。

  • 道路中心始终显示在导入的道路中间。以前，一些道路在道路边缘显示道路中心。

  • 交叉口使用 RoadGroup 对象表示，该对象将交叉口内的道路段组合在一起。以前，交叉口内的每个道路段都是单独表示的。因此，导入的道路网络现在使用更少的道路。

  • 驾驶场景中的道路ID、道路数量、交叉口ID和交叉口数量可能与导入的ASAM OpenDRIVE文件中指定的不匹配。

检查道路

导入的道路网络长度为数千米。使用鼠标滚轮放大道路网络，以更仔细地检查它。道路网络包含一个连接六条道路的环形交叉口。

请验证道路网络是否按预期导入，同时注意应用程序中的以下限制和行为。

ASAM OpenDRIVE导入限制

Driving Scenario Designer 应用程序不支持ASAM OpenDRIVE规范的所有组件。

  • 您只能导入车道、车道类型信息和道路。不支持导入道路对象和交通信号。

  • 包含大型道路网络的ASAM OpenDRIVE文件可能需要几分钟的时间来加载。此外，这些道路网络可能导致应用程序画布上的交互速度变慢。大型道路网络的示例包括模拟城市道路或长度为数千米的道路

  • 不支持具有可变宽度的车道。车道宽度设置为该车道中最大的宽度。例如，如果车道的宽度从2米变化到4米，应用程序将在整个车道上将车道宽度设置为4米。

  • 当您导入具有多个车道规格的单行道时，应用程序仅支持与车道行驶方向匹配的段锥度位置。例如，应用程序仅支持右侧车道的右侧锥度位置。不支持右侧车道的左侧或两种类型的锥度位置。

  • 支持车道类型信息指定为 driving, border, restricted, shoulder, 和 parking 的道路。导入任何其他车道类型信息的车道将被导入为边界车道。

  • 不支持车道标线样式Bott Dots, Curbs, 和 Grass。具有这些标线样式的车道将被导入为未标记的车道。

道路方向

在 Driving Scenario Designer 应用程序中，道路的方向可能与其他显示ASAM OpenDRIVE道路的工具中的道路方向不同。下表显示了应用程序和OpenDRIVE ODR Viewer之间的方向差异。

Driving Scenario Designer OpenDRIVE ODR Viewer

在OpenDRIVE ODR查看器中，X轴沿着查看器的底部运行，而Y轴沿着查看器的左侧运行。

在 Driving Scenario Designer 应用程序中，Y轴沿着画布的底部运行，而X轴沿着画布的左侧运行。该应用程序中的世界坐标系与驾驶场景中车辆使用的车辆坐标系 (XV,YV) 对齐，其中：

  • XV轴（纵向轴）从场景中的车辆向前指向。

  • YV轴（横向轴）指向车辆的左侧，当面向前方时。

要了解更多关于坐标系的详细信息，请参阅 Coordinate Systems in Automated Driving Toolbox。

向场景中添加参与者和传感器

您可以向包含ASAM OpenDRIVE道路的场景中添加参与者和传感器。

在添加角色和传感器之前，如果启用了道路交互，请考虑禁用它们，以防止意外拖动道路中心并更改道路网络。如果启用了道路交互，在 Scenario Canvas 的左下角，单击“配置场景画布”按钮，然后取消选中 Enable road interactions。

通过右键单击画布中的一条道路并选择 Add Car 来向场景中添加自车。要指定汽车的轨迹，请右键单击画布中的汽车，选择 Add Forward Waypoints (Ctrl+F)，并在道路上添加汽车要经过的航点。在添加沿道路的最后一个航点后，按Enter键。汽车会自动旋转到第一个航点的方向。

在应用程序的工具栏上，点击 Add Camera 以向自车添加相机传感器。然后，在传感器画布上，将相机添加到代表汽车前窗的预定义位置。

配置相机以检测车道。在左侧窗格中，切换到 Sensors 选项卡，展开 Detection Parameters 部分。然后，将 Detection Type 参数设置为 Lanes。

生成合成检测结果

要从相机生成车道检测结果，请在应用程序的工具栏上点击 Run。随着场景的运行， Ego-Centric View 将显示从自车角度观察的场景。 Bird’s-Eye Plot 将显示自车的左车道和右车道边界。

要导出生成场景及其检测结果的MATLAB函数，请在应用程序的工具栏上点击 Export > MATLAB Function。

要将检测结果导出到MATLAB工作区，请在应用程序的工具栏上点击 Export > Export Sensor Data。为工作区变量命名并点击 OK。

保存场景

生成检测结果后，点击 Save 以保存场景文件。此外，您还可以将传感器模型保存为单独的文件。您还可以将道路和角色模型一起保存为单独的场景文件。

您可以从应用程序中重新打开此场景文件。或者，您也可以在MATLAB命令提示符下使用以下语法。

drivingScenarioDesigner(scenarioFileName)

如果您正在Simulink中开发驾驶算法，您可以使用 Scenario Reader 模块将道路和角色从 drivingScenario 对象或场景文件读入到您的模型中。包含大型ASAM OpenDRIVE道路网络的场景文件可能需要几分钟的时间才能读入模型。Scenario Reader模块不直接读取传感器数据。要将在应用程序中创建的传感器添加到Simulink模型中，您可以通过选择 Export > Simulink Model 来生成包含您的场景和传感器的模型。在该模型中，Scenario Reader模块读取场景，而雷达和视觉传感器模块对传感器进行建模。
