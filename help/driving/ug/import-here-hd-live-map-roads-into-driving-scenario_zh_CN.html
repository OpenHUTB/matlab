将HERE高清实时地图道路导入驾驶场景

HERE HD Live Map1 (HERE HDLM)，是由HERE Technologies开发的基于云的网络服务，可以让您访问高精度、持续更新的地图数据。使用 Driving Scenario Designer 应用程序，您可以从HERE HDLM服务中导入地图数据，并用它来生成驾驶场景中的道路。

本示例重点介绍在应用程序中导入地图数据的方法。另外，要将HERE HDLM道路导入到 drivingScenario 对象中，请使用 roadNetwork 函数。

设置HERE HDLM凭据

要访问HERE HDLM网络服务，您必须输入来自HERE Technologies的有效凭据。通过使用 hereHDLMCredentials 函数来设置这些凭据。在 MATLAB® 命令提示符下，输入：

hereHDLMCredentials setup

在HERE HD Live Map凭据对话框中，输入有效的 Access Key ID 和 Access Key Secret。要在您的计算机上保存您的凭据，以便在将来的MATLAB会话中使用，请在对话框中选择 Save my credentials between MATLAB sessions，并单击 OK。凭据现在已保存在您的计算机上的MATLAB会话的其余部分中。

如果您需要更改凭据，可以通过使用 hereHDLMCredentials 函数删除它们并设置新的凭据。

指定地理坐标

要选择要导入的道路，您需要指定一个感兴趣的区域，以获取道路数据。为了定义这个感兴趣的区域，请指定接近该道路数据的纬度和经度坐标。您可以为单个点或一组点指定坐标，例如构成驾驶路线的点。

指定驾驶路线的坐标。

  1. 加载一个由经纬度坐标组成的驾驶路线。在MATLAB命令提示符下，输入以下命令：

    data = load('geoSequence.mat');
lat = data.latitude;
lon = data.longitude;

  2. 打开应用程序。

    drivingScenarioDesigner

  3. 在应用程序工具栏上，选择 Import ，然后选择 HERE HD Live Map。如果您之前输入或保存了HERE凭据，则对话框直接打开到您可以指定地理坐标的页面。

  4. 选择 From Workspace 并选择路线坐标的变量。

    • 将 Latitude 参数设置为 lat。

    • 将 Longitude 参数设置为 lon。

    以下表格描述了指定纬度和经度坐标的完整选项列表。

    Specify Geographic Coordinates Parameter Value Description Latitude Parameter Value Longitude Parameter Value
    From Workspace

    在MATLAB工作区中指定一组纬度和经度坐标，例如通过GPS获取的驾驶路线。

    工作区变量包含在[–90, 90] 范围内的十进制值的向量。单位为度。

    纬度和经度必须具有相同的大小。在选择纬度变量后，经度列表只包括与您选择的纬度大小相同的变量。

    工作区变量包含在[-180, 180]范围内的十进制值的向量。单位为度。

    纬度和经度必须具有相同的大小。在选择L经度变量后，如果选择了不同大小的纬度变量，对话框将清除您的经度选择。
    Input Coordinates

    指定一个地理点的纬度和经度坐标。

    		 十进制标量，范围为[–90, 90]。单位为度。 十进制标量，范围为[-180, 180]。单位为度。

选择包含道路的区域

在指定纬度和经度坐标后，对话框框的 Select Region 部分会在地图上以橙色显示这些坐标。还会显示驾驶路线中的第一个坐标作为地理参考点。该点是导入场景的原点。单击此点以显示或隐藏坐标数据。

坐标点连接成一条线。在坐标周围显示一个矩形感兴趣区域。在对话框的下一页中，您可以根据至少部分位于该区域内的道路来选择要导入的道路。

您可以更改该区域的大小或移动它以选择不同的道路。要放大或缩小该区域，请使用地图显示右上角的按钮。

在坐标仍然位于该区域内的情况下，点击 Next。

选择要导入的道路

选择区域后，对话框中的 Select Roads 部分将显示为黑色的可选择道路。

使用所选区域，通过点击 Select Nearest Roads 来选择最靠近驾驶路线的道路。所选道路将与驾驶路线重叠，并显示为蓝色。

以下表格描述了您可以执行的其他选择道路的操作。

Goal Action
选择该地区的个别道路。 点击个别道路以选择它们。
选择该地区的所有道路。 点击 Select All。
从该地区选择除了少数道路之外的所有道路。 点击 Select All，然后点击个别道路以取消选择它们。
选择离指定坐标最近的该地区的道路。 点击 Select Nearest Roads。当您有一系列非稀疏坐标时，请使用此选项。如果您的坐标是稀疏的，或者这些坐标的基础HERE HDLM数据是稀疏的，那么该应用程序可能无法选择最近的道路。
从该地区选择一部分道路，例如该地区上半部分的所有道路。

在地图显示的左上角，点击“选择道路”按钮。然后，绘制一个矩形框选中道路。

  • 要从此选择中取消选择一部分道路，请点击“取消选择道路”按钮。然后，绘制一个矩形框取消选择道路。

  • 要取消选择所有道路并重新开始，请点击 Deselect All。

注意

您选择的道路数量直接影响应用程序的性能。请选择您需要创建驾驶场景所需的最少道路。

导入道路

在选择最接近路线的道路后，点击 Import。应用程序将导入HERE HDLM道路并生成道路网络。

为了保持与地理地图显示的相同对齐，Scenario Canvas 的X轴在底部，Y轴在左侧。在不从地图导入的驾驶场景中，X轴在左侧，Y轴在底部。这种对齐与Automated Driving Toolbox™的世界坐标系一致。

场景的原点对应于地理参考点，并且是驾驶路线中指定的第一个点。即使您从驾驶路线的末端选择道路，原点仍然锚定在这个第一个点上。如果您使用 Input Coordinates 选项指定了一个单独的地理点，则原点是该点。

默认情况下，禁用道路交互。禁用道路交互可以防止意外修改网络，并通过隐藏道路中心来减少视觉杂乱。如果您想修改道路，在 Scenario Canvas 的左下角，点击配置场景画布按钮。然后，选择 Enable road interactions。

注意

在某些情况下，应用程序无法导入所有选择的道路。应用程序会暂停导入，并在对话框中以红色突出显示无法导入的道路。要继续导入所有其他选择的道路，请点击 Continue。

与地图数据比较导入的道路

应用程序中生成的道路网络与实际的HERE HDLM道路网络存在一些差异。例如，实际的HERE HDLM道路网络包含具有不同宽度的道路。Driving Scenario Designer 应用程序不支持此功能。相反，应用程序将每条道路的宽度设置为其整个长度中找到的最大宽度。这种改变增加了道路的宽度，可能导致道路重叠。

有关不受支持的HERE HDLM道路和车道功能的更多详细信息，请参阅 Driving Scenario Designer 应用程序参考页面的 Limitations 部分。

保存场景

保存场景文件。保存场景后，您将无法将其他HERE HDLM道路导入其中。相反，您需要创建一个新的场景并导入新的道路网络。

现在，您可以向场景中添加参与色和传感器，为测试驾驶算法生成合成车道和物体检测，或将场景导入到Simulink®中。
