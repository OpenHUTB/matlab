Driving Scenario Designer中的欧洲新车评估计划（Euro NCAP）驾驶场景

Driving Scenario Designer 应用程序提供了一个预置场景库，代表了欧洲新车评估计划 (Euro NCAP®) 的测试协议。该应用程序包括用于测试自动紧急制动（AEB）、紧急车道保持 (ELK)和车道保持辅助 (LKA) 系统的场景。

Choose a Euro NCAP Scenario

要开始使用，打开 Driving Scenario Designer 应用程序。在MATLAB® 命令提示符下，输入 drivingScenarioDesigner 。

在应用程序中，欧洲新车评估计划（Euro NCAP）的场景以MAT文件的形式存储，并组织成文件夹。要打开一个Euro NCAP文件，在应用程序的工具栏上，选择 Open > Prebuilt Scenario。 PrebuiltScenarios 文件夹将打开，其中包含应用程序中提供的所有预置场景的子文件夹（请参阅 Prebuilt Driving Scenarios in Driving Scenario Designer）。

双击 EuroNCAP 文件夹，然后从这些子文件夹中选择一个Euro NCAP场景。

  • Autonomous Emergency Braking

  • Emergency Lane Keeping

  • Lane Keep Assist

自动紧急制动

这些场景旨在测试自动紧急制动 (AEB) 系统。 AEB 系统会提醒驾驶员即将发生碰撞，并自动施加制动以防止碰撞或减轻碰撞的影响。一些 AEB 系统会为碰撞做好车辆和约束系统的准备。下表描述了 AEB 场景的一个子集。

File Name Description
AEB_Bicyclist_Longitudinal_25width.mat

自车与前方的自行车发生碰撞。在碰撞之前，自行车和自车沿纵向轴向同向行驶。在碰撞时，自行车距离自车宽度的25%处。

其他场景会改变碰撞时自行车的位置。
AEB_CCRb_2_initialGap_12m.mat

一个车对车的后部制动 (CCRb) 场景，自车追尾一个制动的车辆。制动车辆开始以2 m/s2的加速度减速。自车和制动车辆之间的初始间距为 12 米。

其他场景会改变减速度的大小以及自车与制动车辆之间的初始间距。
AEB_CCRm_50overlap.mat

一个车对车的后部移动 (CCRm) 场景，自车追尾一个移动的车辆。在碰撞发生时，自车与移动车辆的宽度重叠了 50% 。

其他场景会改变重叠的程度和重叠的位置。
AEB_CCRs_-75overlap.mat

一种车辆对车辆的后方静止 (CCRs)  场景，即自车追尾一辆静止车辆。在碰撞发生时，自车与静止车辆的宽度重叠部分为静止车辆宽度的-75%。当自车位于另一辆车辆的左侧时，重叠百分比为负值。

其他场景会改变重叠的程度和重叠的位置。
AEB_Pedestrian_Farside_50width.mat

自车与一名从道路左侧行驶的行人发生碰撞，欧洲新车评估计划（Euro NCAP）的测试协议将其称为远侧。这些协议假设车辆在道路的右侧行驶。因此，道路的左侧是离自车最远的一侧。在碰撞发生时，行人距离自车宽度的中心位置为50%。

其他场景会改变碰撞发生时行人的位置。
AEB_PedestrianChild_Nearside_50width.mat

自车与一名从道路右侧行驶的行人发生碰撞，欧洲新车评估计划（Euro NCAP）的测试协议将其称为近侧。这些协议假设车辆在道路的右侧行驶。因此，道路的右侧是离自车最近的一侧。在碰撞发生时，行人距离自车宽度的中心位置为50%。
AEB_PedestrianTurning_Farside_50width.mat

自车在一个交叉口转弯，并与一名与车辆左侧（或远侧）平行行驶的行人发生碰撞，这是模拟开始时的情况。在碰撞发生时，行人距离自车宽度的中心位置为50%。

在另一个情景中，行人位于交叉口的另一侧，并在模拟开始时与车辆的右侧（或近侧）平行行驶。
Reverse_AEB_Pedestrian_Stationary_50width.mat

自车倒车并与一名静止的行人碰撞。在碰撞发生时，行人距离自车宽度的中心位置为50%。

在另一个情景中，在碰撞发生之前，行人从车辆前方参考系的右侧（或近侧）行驶。

紧急车道保持

这些情景是为了测试紧急车道保持（ELK）系统而设计的。ELK系统通过警告驾驶员即将发生的无意识车道偏离，以防止碰撞发生。下表描述了ELK情景的一个子集。

File Name Description
ELK_FasterOvertakingVeh_Intent_Vlat_0.5.mat

自车故意变道并与在另一车道上超车的速度更快的车辆发生碰撞。自车的横向速度为 0.5 m/s。

其他情景会改变横向速度以及车道变更是有意还是无意的。
ELK_OncomingVeh_Vlat_0.3.mat

自车无意中变道并与对向车道上的车辆碰撞。自车的横向速度为 0.3 m/s。

其他情景会改变横向速度。
ELK_OvertakingVeh_Unintent_Vlat_0.3.mat

自车无意中变道，超越了另一车道上的一辆车，并与该车辆发生碰撞。自车的横向速度为 0.3 m/s。

其他情景会改变横向速度以及车道变更是有意还是无意的。
ELK_RoadEdge_NoBndry_Vlat_0.2.mat

自车无意中变道，最终停在道路边缘。道路边缘没有车道边界标线。自车的横向速度为 0.2 m/s。

其他情景会改变横向速度以及道路边缘是否有实线边界、虚线边界或无边界。

车道保持辅助系统（LKA）

这些情景是为了测试车道保持辅助系统（Lane Keep Assist，简称LKA）。LKA系统能够检测到无意的车道偏离，并自动调整车辆的转向角度，以保持在车道边界内。下表列出了LKA情景的一部分。

File Names Description
LKA_DashedLine_Solid_Left_Vlat_0.5.mat

自我车辆无意中从在 左边，右边是实心。汽车从左侧（虚线）离开车道， 以 0.5 m/s 的横向速度行进。

其他情况会改变横向速度以及是否车辆越过的虚线车道位于左侧或 右。
LKA_DashedLine_Unmarked_Right_Vlat_0.5.mat

自我车辆无意中从在右边，左边没有标记。汽车从右侧离开车道（虚线） 侧面，以 0.5 m/s 的横向速度行进。

其他情况会改变横向速度以及是否 车辆越过的虚线车道标记位于左侧或 右。
LKA_RoadEdge_NoBndry_Vlat_0.5.mat

自我车辆无意中偏离车道并最终在 路边。道路边缘没有车道边界标记。汽车在横向速度为 0.5 m/s。

其他情景会改变横向速度。
LKA_RoadEdge_NoMarkings_Vlat_0.5.mat

自车无意中偏离车道，并最终停在道路边缘。道路上没有车道标线。车辆的横向速度为 0.5 m/s。

其他情景会改变横向速度。
LKA_SolidLine_Dashed_Left_Vlat_0.5.mat

自车无意中偏离了左侧为实线、右侧为虚线的车道。车辆从左侧（实线）偏离车道，横向速度为 0.5 m/s。

其他情景会改变横向速度，并且车辆越过的实线车道标记是在左侧还是右侧。
LKA_SolidLine_Unmarked_Right_Vlat_0.5.mat

自车无意中偏离了右侧为实线、左侧没有标线的车道。车辆从右侧（实线）偏离车道，横向速度为 0.5 m/s。

其他情景会改变横向速度，并且车辆越过的实线车道标记可能位于左侧或右侧。

修改场景

在欧洲新车评估计划（Euro NCAP）的默认情景中，自车不包含传感器。如果您要测试车辆传感器，在应用程序的工具栏上，点击 Add Camera 或 Add Radar 以向自车添加传感器。然后，在 Sensor 选项卡上，调整传感器参数以匹配您的传感器型号。如果您要测试摄像头传感器，并使其能够检测车道，展开 Detection Parameters 部分，并将 Detection Type 设置为 Lanes & Objects。

您还可以调整场景中道路和参与者的参数。例如，从左侧的 Actors 选项卡中，您可以更改自车或其他参与者的位置或速度。从 Roads 选项卡中，您可以更改车道的宽度或车道标线的类型。

Generate Synthetic Detections

要从添加的任何传感器生成检测结果，请点击 Run.。当场景运行时，Ego-Centric View 将以自车的视角显示场景，而 Bird’s-Eye Plot 则显示检测结果。

导出检测。

  • 要将传感器数据导出到 MATLAB 工作区，请在应用程序的工具栏上选择 Export > Export Sensor Data。为工作区变量命名，然后点击 OK。应用程序将传感器数据保存为一个结构，其中包含每个时间步的参与者姿态、物体检测和车道检测等传感器数据。

  • 要导出一个生成场景及其传感器数据的MATLAB函数，请选择 Export > Export MATLAB Function。该函数将传感器数据作为结构返回，将场景作为 drivingScenario 对象返回，将传感器模型作为System对象返回。通过修改此函数，您可以创建原始场景的变体。有关此过程的示例，请参阅Create Driving Scenario Variations Programmatically。

保存方案

由于Euro NCAP场景是只读的，将驾驶场景保存到一个新文件夹中。要保存场景文件，请在应用程序的工具栏上选择 Save > Scenario File As。

您可以从应用程序中重新打开此场景文件。或者，您可以在 MATLAB 命令提示符下使用以下语法。

drivingScenarioDesigner(scenarioFileName)
您还可以使用导出的 drivingScenario 对象重新打开场景。在 MATLAB 命令提示符下，使用以下语法，其中 scenario 是导出对象的名称。
drivingScenarioDesigner(scenario)
要重新打开传感器，请使用以下语法，其中 sensors 是一个传感器对象或这些对象的单元数组。
drivingScenarioDesigner(scenario,sensors)
如果您正在Simulink®中开发驾驶算法，您可以使用 Scenario Reader 块将场景文件或 drivingScenario 对象中的道路和参与者读入到您的模型中。该块不直接读取传感器数据。要将您在应用程序中创建的传感器添加到Simulink模型中，可以通过选择 Export > Export Simulink Model 来生成包含您的场景和传感器的模型。在模型中，生成的Scenario Reader块读取场景，而生成的传感器块定义了传感器。
