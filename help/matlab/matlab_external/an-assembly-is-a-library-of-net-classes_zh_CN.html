程序集是一个 .NET 类库

程序集是 .NET 应用程序的构建块；它们构成了部署、版本控制、重用、激活范围和安全权限的基本单元。 程序集是为协同工作并形成功能的逻辑单元而构建的类型和资源的集合。

要使用 .NET 应用程序，需要使其程序集对 MATLAB®可见。如何执行此操作取决于程序集的部署方式（私有部署还是全局部署）。

  • 全局程序集—在应用程序之间共享并安装在称为全局程序集缓存(Global Assembly Cache, GAC)的公共文件夹中。

  • 私有程序集—由单个应用程序使用。

要将全局程序集加载到 MATLAB 中，请使用程序集的短名称，即不带扩展名的文件名。要加载私有程序集，需要程序集的完整路径（带有扩展名的文件夹和文件名）。此信息位于组件的供应商文档中。有关使用产品的信息，请参阅供应商文档。

.NET 5 及更高版本（包括 .NET Core）不支持从 GAC 加载程序集。如果使用程序集的短名称，框架将搜索随 Desktop Runtime 安装的程序集。

对于 .NET 框架，.NET 框架类库中的程序集在启动时可用。当第一次键入 “NET.” 或 “System.” 时，MATLAB 会动态加载这些程序集。

  • mscorlib.dll

  • system.dll

对于 .NET 5 及更高版本，这些程序集在加载 .NET 时可用。

  • Microsoft.Win32.Registry

  • System.Runtime

  • System.Xml

  • System.Diagnostics.Process

  • System.Net.NetworkInformation

  • System.Security.Principal

  • System.Security.Principal.Windows

要使用任何其他 .NET 程序集，请使用 NET.addAssembly 函数加载该程序集。加载程序集后，可以使用程序集定义的类。

笔记

无法在 MATLAB 中卸载程序集。如果修改并重建自己的程序集，则必须重新启动 MATLAB 才能访问更改。
