使用驾驶场景测试闭环ADAS算法

This example uses:

  • Automated Driving Toolbox
  • Simulink
Open Script

该模型展示了如何在 Simulink® 中测试闭环 ADAS（高级驾驶辅助系统）算法。在闭环ADAS算法中，随着仿真的推进，自我车辆由其场景环境的变化控制。

要测试场景，您可以使用从 Driving Scenario Designer 应用程序保存的驾驶场景。在该模型中，您使用 Scenario Reader 模块读取场景，然后在  Bird's-Eye Scope 上直观地验证算法（自主紧急制动系统，AEB）的性能。

检查驾驶场景

该示例使用的驾驶场景基于您可以通过 Driving Scenario Designer 应用程序访问的预构建的欧洲新车评估计划（Euro NCAP）测试协议场景之一。有关这些场景的更多详细信息，请参阅  Euro NCAP Driving Scenarios in Driving Scenario Designer.。

在应用中打开方案文件。

drivingScenarioDesigner('AEB_PedestrianChild_Nearside_50width_overrun.mat')

单击 Run 以模拟方案。在这种情况下，自我车辆与正在过马路的行人相撞。

在本示例中使用的模型中，您使用 AEB 传感器融合算法来检测行人儿童，并测试自我车辆是否及时制动以避免碰撞。

检查模型

该模型实现了Autonomous Emergency Braking with Sensor Fusion 示例中描述的 AEB 算法，并配置为模拟行人碰撞场景。打开模型。

open_system('AEBTestBenchExample')

场景读取器块从指定的Scenario Reader非自我参与者和道路，并输出非自我参与者。自我车辆通过输入端口传递到块中。

场景读取器模块位于Vehicle Environment > Actors and Sensor Simulation子系统中。打开此子系统。


open_system('AEBTestBenchExample/Vehicle and Environment/Actors and Sensor Simulation')

在 Scenario Reader 模块中，Driving Scenario Designer 文件名参数指定场景文件的名称。您可以指定一个位于 MATLAB® 搜索路径上的场景文件，例如此示例中使用的场景文件，或者指定场景文件的完整路径。或者，您可以通过将驾驶场景的 Source 设置为 From workspace，并将 MATLAB 或模型工作区变量名称设置为有效的 drivingScenario 对象工作区变量的名称来指定 drivingScenario 对象。在闭环仿真中，指定 drivingScenario对象非常有用，因为它使您可以更精细地控制模型中自车的初始位置。

Scenario Reader 模块输出场景中非自车角色的姿态。这些姿态被传递给视觉和雷达传感器，其检测结果用于确定 AEB 控制器的行为。

角色姿态以车辆坐标系输出，其中：

  • X轴从自我车辆指向前方。

  • Y轴指向自我车辆的左侧。

  • 原点位于自我车辆后轴的中心。

尽管该场景包含一个预定义的自车，但 Scenario Reader 模块被配置为忽略该自车定义。相反，自车在模型中定义，并作为输入传递给 Scenario Reader 模块（Source of ego vehicle参数设置为Input port）。随着仿真的进行，AEB 算法确定自车的姿态和轨迹。如果您正在开发一个开环算法，其中自车在驾驶场景中预定义，将Source of ego vehicle参数设置为Scenario。有关示例，请参阅Test Open-Loop ADAS Algorithm Using Driving Scenario.。

可视化仿真

若要可视化方案，请使用Bird's-Eye Scope。在 Simulink 工具之旅的 Review Results下，单击 Bird's-Eye Scope 。然后，在scope,中，单击 Find Signals 并运行模拟。使用AEB算法，自我车辆及时制动以避免碰撞。
